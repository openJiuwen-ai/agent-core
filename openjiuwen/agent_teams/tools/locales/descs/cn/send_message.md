向团队成员发送消息。用于通知成员领取任务、回复进度汇报、升级阻塞问题或协调成员间依赖。

| `to` 值 | 含义 |
|---|---|
| 成员名称 | 点对点消息（DM / direct message / 私聊）— 仅你与该成员两人可见，类似 IM 工具里的"私聊"，其他成员（包括 leader）都收不到。多人需同步同一内容时改用多播或广播 |
| 成员名称数组（如 `["m1","m2"]`） | 多播 — 同一份内容作为多条独立的点对点消息分别发给每个成员，每条独立 read 状态。**开销与接收人数线性增长（每多一个成员都多写一条消息、多发一个事件），同样人数下比广播更贵**，仅当确实需要让团队部分成员同步看到同一内容、且不适合改用广播时才使用，谨慎选择。**当接收人恰好覆盖团队全体其他成员（你之外的所有成员）时，禁止使用多播，必须改用广播 `"*"`——多播覆盖全员只是更贵的广播，工具会直接拒绝这种请求。**不能与 `"*"`/`"user"` 混用；只有全部成员都成功才视为整体成功，部分失败时返回值会列出已送达和失败的成员，不要对已送达的成员重发 |
| `"user"` | 仅 teammate 使用 — 当收到的消息来源（`from`）为 `user` 时，必须用本工具把答复发回 `user`，teammate 的普通文本输出不会送达用户。leader 不要用此值，leader 的所有文本输出用户都会直接看到 |
| `"*"` | 广播给所有成员（团队频道 / channel 消息）— 类似 IM 工具里发到团队公共频道，所有成员都能看到，开销与团队规模成正比，仅用于宣布全局决策、约束变更或需要所有人知晓的信息 |

teammate 的普通文本输出对其他成员和用户都不可见 — 要通信必须调用此工具。leader 的普通文本输出用户可直接看到，无需调用本工具回复用户。来自队友的消息会自动送达，无需轮询。用名称称呼成员，不要使用内部 ID。转发消息时不要引用原文 — 原文已经渲染给了用户。
