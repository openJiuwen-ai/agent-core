审批或拒绝成员提交的执行计划。**仅在团队 `teammate_mode=plan_mode` 时暴露给 Leader**；build_mode 下成员不提交计划，此工具不会出现在工具列表中。

Leader 调用后，系统会向该成员发送审批结果；若批准，其已认领的任务会进入可实施状态（plan_approved）；若拒绝，成员应根据反馈修改后重新提交。

| 参数 | 用法 |
|---|---|
| **member_name** | 提交计划的成员 member_name，应指向当前团队中的有效成员 |
| **approved** | 是否批准该成员当前计划。true 表示批准进入实施，false 表示拒绝并要求修改 |
| **feedback** | 审批反馈。拒绝时应明确指出问题和修改方向；批准时也可给补充要求或提醒 |

适用场景：
- 成员提交计划后，Leader 判断其是否可以进入实施阶段
- 计划方向正确但需要补充约束时，可批准并附带提醒
- 计划存在方向错误、边界不清、依赖遗漏或验收风险时，应拒绝并要求修改

注意：
- 批准不等于任务已完成，而是允许该成员按计划继续实施
- 拒绝时不要只说"重写"或"再想想"，应给出具体修改方向
- 若成员当前没有待审批计划或没有已认领任务，批准可能不会推进任何任务状态
