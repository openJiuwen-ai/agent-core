# 你的任务
请基于上述上下文，提取给定实体之间的**所有事实关系**。

仅提取符合以下条件的事实：
- 涉及实体列表中的**两个不同实体**，可表示为知识图谱中的边
- 事实类型提供了最重要的事实类型列表，优先提取这些类型的事实
- 事实类型不是详尽列表，即使不符合事实类型之一，也要从消息中提取所有事实

# 提取规则
1. 主体和客体必须均来自实体列表，实体ID为{{id_range}}的整数。
2. 每个事实必须连接两个不同的实体。
3. `name`使用**英文大写蛇形命名**（如：FOUNDED, WORKS_AT）。
4. 避免输出重复或语义等价的事实。
5. `content`应为上下文中原始语句的直接引用或简要复述。
6. 使用`当前UTC时间`解析模糊或相对的时间表达（如“上周五”）。
7. **禁止**胡乱猜测上下文中未含括的时间信息。

# 时间提取规则
- 使用**ISO 8601 格式**（如：`2025-09-15T13:49:08+08:00` 或 `2025-09-15T05:49:08`）。
- 已提供的时区信息仅供参考，这不一定是事实。
- 若能合理推测时区，则补充（如：中国事件 → `+08:00`）。
- 若事实有明确的生效时间 → 设为`valid_since`。
- 若事实有明确的失效时间 → 设为`valid_until`。
- 若无明确时效信息 → 两个字段均为`null`。
- 仅提及日期时，时间默认使用`00:00:00`。
- 仅提及年份时，使用该年的1月1日（`01-01T00:00:00`）。{{extra_message}}
