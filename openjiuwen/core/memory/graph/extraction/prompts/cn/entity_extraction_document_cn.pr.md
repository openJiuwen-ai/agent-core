# 你的任务
请基于上述上下文，从**上下文信息**中提取并分类所有**直接或间接**出现的实体。

## 注意事项
1. **参与者提取**
   - 对话参与者必须始终提取为实体节点。
   - 如果同一参与者在消息中多次出现，应合并为同一实体。
   - 仅基于明确事实提取，不做无根据推测。

2. **实体识别**
   - 仅提取上下文信息中**直接或间接**提到的实体（包括被代词/称谓指代的实体）。
   - 不要提取未在上下文信息中被引用的实体。

3. **实体分类**
   - 使用给定的**实体类型定义**描述，为每个实体分配最合适的`entity_type_id`。

4. **事件提取**
   - 尽量将与人类用户相关的事件提取为实体。
   - 事件实体的命名需要足够独特，这样便于区分。

5. **排除规则**
   - 不要提取动作或关系作为实体。
   - 不要提取日期、时间或其他时间信息（这些由其他模块处理）。

6. **消歧与命名**
   - 对代词或模糊指代（如“user/assistant/他/她/它/老李/小张/这个/那些”）进行消歧，并替换为正确的实体名称。
   - 命名必须明确、唯一，不含歧义（如能用全名则使用全名）。{{extra_message}}
