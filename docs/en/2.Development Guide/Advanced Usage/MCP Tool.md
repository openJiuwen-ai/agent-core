# MCP Tool

MCP (Model Context Protocol) is a protocol for communication between AI agents and external tools/services. The openjiuwen framework provides a complete MCP client implementation that supports multiple transport protocols, enabling agents to invoke any tool service conforming to the MCP protocol.

## 1. Core Capabilities

| Capability | Description |
|------------|-------------|
| **Multi-Transport Protocol** | Supports three standard protocols: SSE, Stdio, and Streamable-HTTP. |
| **Local Process Integration** | Communicates with local MCP Servers via Stdio. |
| **Remote Service Integration** | Connects to remote MCP services via SSE/Streamable-HTTP. |
| **OpenAPI Conversion** | Automatically converts OpenAPI specifications to MCP tools. |
| **Playwright Support** | Provides a browser automation MCP client. |
| **Authentication Mechanism** | Supports Header/Query parameter authentication. |
| **Tool Discovery** | Automatically lists and registers all tools provided by MCP servers. |
| **Lifecycle Management** | Handles the addition, refresh, and removal of MCP servers. |
| **Tool Caching** | Supports `expiry_time` expiration mechanism to periodically refresh the tool list. |

## 3. Quick Integration Guide

### 3.1 Creating a Simple MCP Server Example

This section demonstrates a minimal MCP Server: a weather query service. This service exposes a `get_weather` tool that an Agent can invoke via natural language to query weather for any city.

**Implemented Features:**
- Tool name: `get_weather`
- Input parameter: `city` (string, city name)
- Return content: Weather description and temperature

**Code Example:**

```python
# server.py
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("my-server")

@mcp.tool()
def get_weather(city: str) -> str:
    """Get the weather for a city"""
    return f"The weather in {city} is sunny, 25°C"

if __name__ == "__main__":
    mcp.run(transport="stdio")  # Use stdio transport
```

### 3.2 Configuring and Invoking MCP Tools in an Agent

This section demonstrates how to configure and invoke the MCP Server created in 3.1 within a DeepAgent. By passing `McpServerConfig` in `DeepAgentConfig`, the Agent will automatically connect to the MCP server and register its tools upon startup.

**Implemented Features:**
- Configure MCP server connection parameters (server_id, server_name, server_path, client_type)
- Mount MCP server to Agent
- Let Agent invoke MCP tools via natural language

**Code Example:**

```python
# agent_example.py
import asyncio
from openjiuwen.core.foundation.tool import McpServerConfig
from openjiuwen.harness.deep_agent import DeepAgent
from openjiuwen.harness.schema.config import DeepAgentConfig
from openjiuwen.core.single_agent.schema.agent_card import AgentCard

# Note: The Model needs to be configured in advance. Please refer to the standard creation flow of ModelClientConfig + ModelRequestConfig + Model.

async def main():
    # Configure MCP server (corresponding to the local MCP Server started in 3.1)
    mcp_config = McpServerConfig(
        server_id="my-weather-server",
        server_name="my-server",
        server_path="python server.py",  # stdio transport, pointing to server.py from 3.1
        client_type="stdio",
    )

    # Create Agent config and mount MCP server to it
    agent_config = DeepAgentConfig(
        model=model,  # Pass the pre-configured Model instance here
        card=AgentCard(name="assistant", description="Assistant"),
        mcps=[mcp_config],  # Pass MCP config list
    )

    agent = DeepAgent(agent_config.card).configure(agent_config)

    # After the Agent starts, it will:
    # 1. Automatically connect to MCP server (start server.py process via stdio)
    # 2. Call list_tools() to get the tool list (get_weather)
    # 3. Register tools as available Agent capabilities

    result = await agent.invoke("How's the weather in Beijing today?")
    print(result)

asyncio.run(main())
```


## 4. Transport Protocol Configuration Guide

### McpServerConfig Core Fields

McpServerConfig contains the following core configuration fields:

| Field | Type | Description |
|-------|------|-------------|
| `server_id` | str | Server unique identifier (auto-generated by default) |
| `server_name` | str | Server name |
| `server_path` | str | Server path/address/file path |
| `client_type` | str | Transport type: sse/stdio/streamable-http/openapi/playwright |
| `params` | dict | Extra parameters (e.g., command/args for StdioServerParameters) |
| `auth_headers` | dict | HTTP authentication Header (SSE/StreamableHttp only) |
| `auth_query_params` | dict[str, str] | HTTP authentication Query parameters |

### 4.1 SSE-Based Remote Service Integration

SSE (Server-Sent Events) is suitable for remote MCP services that can establish persistent HTTP connections.

```python
# SSE configuration example
mcp_config = McpServerConfig(
    server_id="remote-weather-api",
    server_name="weather-service",
    server_path="https://api.example.com/mcp/sse",  # SSE endpoint
    client_type="sse",
    auth_headers={"Authorization": "Bearer xxx"},    # Optional: authentication
)
```

### 4.2 Stdio-Based Local Process Integration

Stdio is suitable for local process communication, spawning subprocesses and communicating via standard input/output.

```python
# stdio configuration example
mcp_config = McpServerConfig(
    server_id="local-git-server",
    server_name="git-tools",
    server_path="python /path/to/mcp-git-server.py",  # Local command
    client_type="stdio",
    # params={"args": ["--verbose"]},  # Optional: pass extra arguments to the process
)
```

### 4.3 Streamable-HTTP Based Integration

Streamable-HTTP is suitable for MCP services that support streaming HTTP.

```python
# Streamable-HTTP configuration example
mcp_config = McpServerConfig(
    server_id="http-mcp-server",
    server_name="http-tools",
    server_path="https://api.example.com/mcp/streamable",
    client_type="streamable-http",
    auth_headers={"X-API-Key": "xxx"},  # Authentication header
)
```

### 4.4 OpenAPI Conversion Client

Convert existing OpenAPI specifications into MCP tools:

```python
# OpenAPI configuration example
mcp_config = McpServerConfig(
    server_id="openapi-converter",
    server_name="openapi-tools",
    server_path="openapi.json",
    client_type="openapi",
)
```

### 4.5 Playwright Browser Automation

Playwright is suitable for MCP services requiring browser automation, such as web scraping and UI testing scenarios.

```python
# Playwright configuration example
mcp_config = McpServerConfig(
    server_id="playwright-browser",
    server_name="browser-automation",
    server_path="http://localhost:9222",  # Playwright MCP Server address
    client_type="playwright",
)
```

## 5. Complete Demo: "Best Practices" for Enterprise Customers

### Scenario: Banking Financial Advisory Agent

This section demonstrates a financial industry scenario: a bank's AI-powered customer service Agent invokes backend wealth management system via MCP tools, helping users query financial product information and complete purchases.

**Business Background:**
- Users want to query financial products and compare yields via natural language
- Users want to complete financial product purchases directly through dialogue
- Bank backend systems provide service capabilities as MCP tools

**Implemented Features:**
- Query financial products (name, return rate, risk level, minimum investment)
- Purchase specified financial products (input product name and amount)

**Code Example:**

```python
# financial_agent_demo.py
import asyncio
from openjiuwen.core.foundation.tool import McpServerConfig
from openjiuwen.harness.deep_agent import DeepAgent
from openjiuwen.harness.schema.config import DeepAgentConfig
from openjiuwen.core.single_agent.schema.agent_card import AgentCard

async def financial_agent_demo():
    # MCP server list (simulating the bank's wealth management system)
    mcps = [
        # Financial product query service (local stdio)
        McpServerConfig(
            server_id="fin-product-query",
            server_name="financial-product-service",
            server_path="python fin_product_server.py",
            client_type="stdio",
        ),
        # Financial product purchase service (remote SSE)
        McpServerConfig(
            server_id="fin-product-purchase",
            server_name="financial-purchase-service",
            server_path="https://api.bank.com/mcp/sse/purchase",
            client_type="sse",
            auth_headers={"X-Bank-Token": "bank-auth-token"},
        ),
    ]

    # Configure Agent (Model needs to be pre-configured)
    agent_config = DeepAgentConfig(
        model=model,  # Pass the pre-configured Model instance here
        card=AgentCard(
            name="bank-assistant",
            description="Banking Financial Assistant: Query financial products, complete purchases"
        ),
        mcps=mcps,
        max_iterations=15,
    )

    agent = DeepAgent(agent_config.card).configure(agent_config)

    # Natural language invocation of MCP tools
    tasks = [
        "What stable financial products are available now and what are the returns?",
        "I want to purchase 500,000 yuan of Stable Profit Treasure financial product",
    ]

    for task in tasks:
        print(f"\nTask: {task}")
        result = await agent.invoke(task)
        print(f"Result: {result}")

asyncio.run(financial_agent_demo())
```

## 6. Troubleshooting FAQ

### Q1: MCP Server Connection Failure

**Symptoms:** Connection timeout or unable to establish connection when the Agent starts.

**Troubleshooting Steps:**
1. Check if `server_path` is accessible (test by opening URL directly in browser)
2. Confirm `client_type` matches server transport type (stdio for stdio, sse for sse)
3. Check network connectivity (firewall, proxy, VPN)
4. Confirm MCP Server service is started

**Example:**
```python
# Troubleshooting: Confirm server_path is accessible
# SSE scenario
server_path = "https://api.example.com/mcp/sse"  # e.g., test by opening the URL directly in a browser

# stdio scenario
server_path = "python /path/to/server.py"  # Manually run to confirm no syntax errors
```

### Q2: Authentication Token Expired or Invalid

**Symptoms:** MCP Server returns 401/403 error, or tool invocation returns authentication failure.

**Troubleshooting Steps:**
1. Check if the Token in `auth_headers` is correct.
2. Confirm the Token has not expired.
3. Check if Token format matches server requirements (e.g., `Bearer xxx` vs. just `xxx`)

**Example:**
```python
# Incorrect: Token may have expired
auth_headers={"Authorization": "Bearer old-token"}

# Correct: Use environment variables to fetch the latest Token
import os
auth_headers={"Authorization": f"Bearer {os.getenv('MCP_API_TOKEN')}"}
```

### Q3: How to Dynamically Refresh MCP Tool List

**Symptoms:** MCP Server added or removed tools at runtime, but the Agent still sees the outdated tool list.

**Troubleshooting Steps:**
1. Confirm the MCP Server has correctly started and registered target tools
2. Call the ResourceManager's refresh interface to re-fetch the tool list.
3. After refresh, verify new tools are registered

**Example:**
```python
from openjiuwen.core.runner import Runner

# Refresh by server_id
result = await Runner.resource_mgr.refresh_mcp_server(server_id="my-server")
print(f"Refresh result: {result}")

# Or refresh by name
result = await Runner.resource_mgr.refresh_mcp_server(server_name="my-server")

# View current tool list after refresh
tools = Runner.resource_mgr.get_tool_ids()
print(f"Current tools: {tools}")
```

### Q4: Tool Invocation Returns Unexpected Results

**Symptoms:** Tool invocation succeeds, but returned result is empty, format is abnormal, or does not match expectations.

**Troubleshooting Steps:**
1. Check if the data format returned by MCP Server matches expectations
2. Confirm `inputSchema` parameter definition matches actual passed parameters
3. Check MCP Server logs to determine if issue is on Server side or Client side
4. If using a network proxy, confirm the proxy has not modified the response payload.

**Example:**
```python
# Add logs at invocation site to print full return
result = await agent.invoke("Query weather")
print(f"Raw return: {result}")  # Confirm the exact return content

# Check if it's a parameter issue
# MCP Server's inputSchema should match the Agent's parameter payload
```
