# openjiuwen.core.single_agent.interrupt

Interrupt request data structure module. This module defines data structures used during tool execution interruption, primarily in two scenarios: parsing interrupt information returned by the Agent, and creating interrupt requests when implementing custom Rails.

## class InterruptRequest

Interrupt request data structure that defines the data format required from user input during an interruption.

**Parameters**:

* **message**(str): Prompt message to explain to the user what input is needed. Default: `""`.
* **payload_schema**(dict): Data structure definition for user input, following JSON Schema specification. Default: `{}`.
* **auto_confirm_key**(str): Auto-confirmation key, typically the tool name, used for auto-confirmation configuration. Default: `""`.
* **questions**(List[dict], optional): Structured question list for displaying complete question information in multi-question mode (including header, question, options, multi_select, preview, etc.). Default: `None`.

## class ToolCallInterruptRequest

Interrupt request data structure with tool call context, inherits from [InterruptRequest](#class-interruptrequest), used to display detailed interrupt information to the user.

**Parameters**:

* **message**(str): Prompt message to explain to the user what input is needed. Default: `""`.
* **payload_schema**(dict): Data structure definition for user input, following JSON Schema specification. Default: `{}`.
* **auto_confirm_key**(str): Auto-confirmation key, typically the tool name. Default: `""`.
* **questions**(List[dict], optional): Structured question list (inherited from [InterruptRequest](#class-interruptrequest)). Default: `None`.
* **tool_name**(str): Name of the tool that triggered the interruption. Default: `""`.
* **tool_call_id**(str): Unique identifier for the tool call, generated by the LLM. Default: `""`.
* **tool_args**(Any): Tool call arguments. Default: `None`.
* **index**(int, optional): Index during concurrent calls. Default: `None`.

### classmethod from_tool_call

```python
classmethod from_tool_call(request: InterruptRequest, tool_call: Any) -> ToolCallInterruptRequest
```

Creates a ToolCallInterruptRequest instance from an InterruptRequest and ToolCall.

**Parameters**:

* **request**([InterruptRequest](#class-interruptrequest)): Interrupt request data structure.
* **tool_call**(Any): Tool call object, must contain `name`, `id`, `arguments`, and `index` attributes.

**Returns**:

**ToolCallInterruptRequest**, an interrupt request data structure containing tool call context.

**Example**:

When an interruption is triggered during Agent execution, the returned result structure is as follows:

```python
{
    'result_type': 'interrupt',
    'state': [
        OutputSchema(
            type='__interaction__',
            index=0,
            payload=InteractionOutput(
                id='call_dd26eaa14529440c81b54eab',
                value=ToolCallInterruptRequest(
                    message='Please approve or reject?',
                    payload_schema={
                        'description': 'Payload for user confirmation response.',
                        'properties': {
                            'approved': {'title': 'Approved', 'type': 'boolean'},
                            'feedback': {'default': '', 'title': 'Feedback', 'type': 'string'},
                            'auto_confirm': {'default': False, 'title': 'Auto Confirm', 'type': 'boolean'}
                        },
                        'required': ['approved'],
                        'title': 'ConfirmPayload',
                        'type': 'object'
                    },
                    auto_confirm_key='read',
                    tool_name='read',
                    tool_call_id='call_dd26eaa14529440c81b54eab',
                    tool_args='{"filepath": "/tmp/test1.txt"}',
                    index=0
                )
            )
        )
    ],
    'interrupt_ids': ['call_dd26eaa14529440c81b54eab']
}
```

Developers can retrieve interrupt information in the following way:

```python
# Get ToolCallInterruptRequest object
interrupt_output = result['state'][0]  # OutputSchema
request = interrupt_output.payload.value  # ToolCallInterruptRequest

# Get interrupt details
print(f"Tool name: {request.tool_name}")
print(f"Tool call ID: {request.tool_call_id}")
print(f"Tool arguments: {request.tool_args}")
print(f"Prompt message: {request.message}")
print(f"payload_schema: {request.payload_schema}")
```

> **Usage Scenarios**
>
> The classes in this module are used in the following two scenarios:
>
> 1. **Parsing Interrupt Output**: When an interruption is triggered during Agent execution, the returned result contains a `ToolCallInterruptRequest` object, from which developers can retrieve detailed interrupt information (such as tool name, tool arguments, etc.).
>
> 2. **Custom Rail**: When implementing custom Rails, developers need to create `InterruptRequest` objects to define the data structure for interrupt requests, including prompt messages and the data format for user input.
>
> For complete usage instructions, please refer to the [Interrupt Rail documentation](../../openjiuwen.harness/rails/interrupt.md).
