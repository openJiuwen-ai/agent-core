# A2A 客户端

`A2AClient` 是对 A2A SDK Client 的 openjiuwen 封装。
它接收 openjiuwen 风格的输入，并转换为 A2A 请求。

## 主要 API

```python
from openjiuwen.extensions.a2a.a2a_client import A2AClient
```

- `A2AClient(card: AgentCard | None = None, polling: bool = False)`
  - 使用 A2A `AgentCard` 创建客户端。
  - `polling` 会透传到到底层 A2A SDK Client 配置。
  - 这个封装会在构造时完成 SDK 客户端初始化。
- `await client.invoke(inputs: dict[str, Any]) -> AgentResult`
  - 发送请求，返回首个可用事件对应的 `AgentResult`。
  - 这与 SDK 的 `send_message()` 迭代器模型一致，因为 A2A SDK 没有单独的 HTTP “invoke” RPC。
- `async for chunk in client.stream(inputs: dict[str, Any])`
  - 发送流式请求，按事件逐条返回 `AgentResult` chunk。
- `await client.stop()`
  - 关闭底层 A2A 客户端连接。

## 入参约定

客户端接收 openjiuwen 风格的请求字典，常见键包括：

- `query`：用户输入文本
- `conversation_id`：openjiuwen Runner 层更常见的会话标识
- `sessionId`：兼容输入写法，会被归一成同一个结果会话值
- 其他元数据字段会在可能时透传
- `conversation_id` 是更推荐的上游会话键，`sessionId` 仅作为兼容输入保留
- `task_id` 不会在客户端侧创建，通常由远端 A2A 服务生成

请求会先交给 `A2ATransformer.to_a2a_request()` 转成 A2A 请求。

## 出参约定

`invoke()` 和 `stream()` 返回的都是 openjiuwen `AgentResult`，A2A 响应会通过 `A2ATransformer.from_a2a_response()` 转回 openjiuwen 数据结构。
返回结果里的 `sessionId` 会优先回填为调用方传入的 `conversation_id`；`sessionId` 仅作为兼容输入保留。
`invoke()` 返回首个可用事件，`stream()` 则按事件流逐条返回。

## 载荷与卡片映射

`A2AClient` 依赖两个转换辅助类：

- `A2ATransformer`：负责请求和响应的转换
- `A2AAgentCardAdapter`：负责把本地 openjiuwen 卡片转换成 A2A 卡片

### 请求映射

| openjiuwen 输入字段 | A2A 字段 | 说明 |
| --- | --- | --- |
| `query` | `message.parts[0].text` | 第一个文本 part 会成为用户输入内容。 |
| `conversation_id` / `sessionId` | `message.context_id` | 会话标识会归一化后写入 A2A 上下文。 |
| 其他顶层字段 | `send_request.metadata` | 除 `query`、`conversation_id` 和 `sessionId` 外的字段会尽量保留到 A2A metadata。 |
| `metadata` | `send_request.metadata` | 如果提供的是 dict，会合并到 A2A metadata。 |

### 响应映射

| A2A 来源对象 / 字段 | openjiuwen `AgentResult` 字段 | 说明 |
| --- | --- | --- |
| `Message.task_id` | `task_id` | 作为结果任务号保留。 |
| `Message.context_id` | `sessionId` | 作为结果会话号保留。 |
| `Message.parts` | `artifacts[0].parts` | 消息内容会转成一个名为 `message` 的 artifact。 |
| `Task.id` | `task_id` | 作为结果任务号保留。 |
| `Task.context_id` | `sessionId` | 作为结果会话号保留。 |
| `Task.status.state` | `status` | 映射为 openjiuwen 的 `TaskStatus`。 |
| `Task.artifacts` | `artifacts` | 按 artifact 逐个转换。 |
| `TaskStatusUpdateEvent` / `TaskArtifactUpdateEvent` | `AgentResult` | 流式事件会转换成增量的 openjiuwen 结果。 |

### AgentCard 映射

| openjiuwen `AgentCard` 字段 | A2A `AgentCard` 字段 | 说明 |
| --- | --- | --- |
| `name` | `name` | 直接复制。 |
| `description` | `description` | 直接复制，若有输入/输出参数会追加到说明文本中。 |
| `input_params` | `description` | 序列化后追加为 `[input_params] ...`。 |
| `output_params` | `description` | 序列化后追加为 `[output_params] ...`。 |
| `interface_url` | `supported_interfaces[0].url` | 当未显式传入 `supported_interfaces` 时使用。 |
| `protocol_binding` | `supported_interfaces[0].protocol_binding` | 默认 `HTTP+JSON`，可被覆盖。 |
| `protocol_version` | `supported_interfaces[0].protocol_version` | 默认 `1.0`，可被覆盖。 |
| `tenant` | `supported_interfaces[0].tenant` | 提供时会保留。 |
| `supported_interfaces` | `supported_interfaces` | 显式接口列表优先于 `interface_url`。 |

### Part 映射

| openjiuwen `Part` 字段 | A2A `Part` 字段 | 说明 |
| --- | --- | --- |
| `text` | `text` | 直接复制。 |
| `raw` | `raw` | 直接复制。 |
| `url` | `url` | 直接复制。 |
| `data`（dict） | `data.struct_value` | 字典会序列化为 protobuf struct。 |
| `data`（其他类型） | `data.string_value` | 非 dict 值会先转成字符串。 |
| `filename` | `filename` | 直接复制。 |
| `media_type` | `media_type` | 直接复制。 |
| `metadata` | `metadata` | 序列化为 protobuf struct。 |

## 插件使用

在基于 runner 的集成里，通常不会直接实例化 `A2AClient`，而是通过远程智能体协议接入：

- `RemoteAgent(protocol=ProtocolEnum.A2A, ...)` 会解析到 `A2A` 远程客户端插件
- `openjiuwen.extensions.a2a` 会把工厂注册到 runner 注册表
- 使用 `pip install "openjiuwen[all-a2a]"` 或 `uv sync --extra all-a2a` 可以安装 A2A SDK 依赖包

如果你需要在 `RemoteAgent` 之外直接使用客户端，也可以手动构造 `A2AClient` 并传入 A2A `AgentCard`。

## 直接按协议使用

当你想显式创建 A2A remote agent 时，可以这样写：

```python
from openjiuwen.core.runner.drunner.remote_client.remote_agent import RemoteAgent
from openjiuwen.core.runner.drunner.remote_client.remote_client_config import ProtocolEnum
from openjiuwen.core.single_agent.schema.agent_card import AgentCard

agent = RemoteAgent(
    agent_id="a2a-agent",
    protocol=ProtocolEnum.A2A,
    config={
        "url": "http://127.0.0.1:41241",
        "kwargs": {
            "card": AgentCard(id="a2a-agent", name="a2a-agent"),
            "polling": True,
        },
    },
)
```

这条链路是：

`RemoteAgent -> create_remote_client("A2A") -> A2ARemoteClient -> A2AClient -> A2A SDK`

### 参数说明

| 参数 | 含义 |
| --- | --- |
| `card` | 远程客户端路径下必传，这里传入的是 openjiuwen 的 `AgentCard`，随后会被转换成 A2A 卡片。 |
| `polling` | 可选，默认值为 `False`，会透传到 A2A SDK Client 配置。 |

### polling 行为

`polling` 用来控制底层 A2A SDK 是否在发送消息后自动轮询任务。

| 设置 | 行为 | 返回结果 |
| --- | --- | --- |
| `polling=True` | 客户端发送请求后持续轮询 A2A 任务，直到进入终态。 | 通常返回最终结果，比如完成态的 `Message` 或 `Artifact`。 |
| `polling=False` | 客户端发送请求后不自动轮询。 | 先返回一个任务相关结果，生命周期需要你自己接管。 |

如果你手动接管任务生命周期，可以根据返回的任务标识继续查询任务状态，或者用 SDK 的任务订阅接口重新订阅任务流。

## 示例

```python
from openjiuwen.extensions.a2a.a2a_client import A2AClient

client = A2AClient(card=a2a_agent_card)
result = await client.invoke({"query": "hello", "conversation_id": "conv-1"})
```
