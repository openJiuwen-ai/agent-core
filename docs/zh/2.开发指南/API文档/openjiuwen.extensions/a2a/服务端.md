# A2A 服务端

`A2AServer` 和 `A2AServerAdapter` 用于把 openjiuwen Agent 暴露成 A2A 服务。
这一页只保留启动方式和请求流转。
使用前请先安装 `openjiuwen[all-a2a]`。

## 启动方式

```python
from openjiuwen.extensions.a2a.a2a_server import A2AServer
from openjiuwen.extensions.a2a.a2a_server_adapter import A2AServerAdapter
```

`A2AServer` 可以直接启动：

```python
server = A2AServer(agent_card=agent_card, interface_url="http://127.0.0.1:8000/a2a/jsonrpc/")
await server.start()
```

`A2AServerAdapter` 是 runner 侧桥接层。`RunnerConfig.enable_a2a=True` 时，`AgentAdapter`
会通过 `create_server_adapter("A2A", ...)` 解析它，并为本地 agent 启动 A2A 服务。

## 标准路径

这是框架内暴露本地 agent 为 A2A 的标准路径：

1. 设置 `distributed_mode=True`，否则 `AgentMgr.add_agent()` 不会创建 `AgentAdapter`
2. 设置 `enable_a2a=True`，否则 `AgentAdapter` 不会创建 A2A 服务适配器
3. 调用 `add_agent(..., card=AgentCard(...), interface_url=...)`；`card` 必填，`interface_url`
   可以覆盖卡片上的同名配置

当 `AgentAdapter.start()` 运行时，会先启动 MQ server；如果启用了 A2A，还会继续启动
`A2AServerAdapter`。如果 `interface_url` 能解析出 host 和 port，适配器还会在后台启动
uvicorn 线程运行 A2A 服务。

`AgentAdapter` 里的 A2A handler 直接绑定到 `Runner.run_agent` 和 `Runner.run_agent_streaming`，
所以 A2A 服务会回调到同一个 `agent_id` 下注册的 agent。

## 请求流转

A2A 请求进入服务端后，会依次经过：

1. A2A SDK 请求处理器
2. `A2AAgentExecutor`
3. `A2ATransformer`
4. openjiuwen 业务逻辑
5. `TaskUpdater` 事件回写

`A2AAgentExecutor` 会先写入 `Task` 快照，再发送状态更新事件，这样当前 a2a-sdk 客户端才能正确消费流式结果。
