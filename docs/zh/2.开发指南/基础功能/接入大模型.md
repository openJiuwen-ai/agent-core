# 接入大模型

不同模型在推理能力、对话流畅性、多轮交互等方面各有特色，用户可根据具体的应用场景灵活选择最适合的模型，例如在需要复杂逻辑推理的场景下选择推理能力更强的模型，在追求自然对话体验的场景下选择交互更自然的模型。

openJiuwen通过 `openjiuwen.core.foundation.llm` 提供统一的**模型客户端 + 配置**体系，推荐的接入方式是：

- 使用 `ModelClientConfig` 描述「如何连接模型服务」（client_provider/api_base/api_key/custom_headers/SSL 等）；
- 使用 `ModelRequestConfig` 描述「这次调用用哪个模型 + 调用参数」（model/temperature/top_p 等）；
- 使用统一入口类 `Model`，通过 `invoke/stream` 进行调用。框架会根据 `client_provider` 自动选择对应的模型客户端实现。

## 使用 Model 调用模型

### 初始化模型

以硅基流动厂商的模型接入为例：

```python
from openjiuwen.core.foundation.llm import (
    Model,
    ModelClientConfig,
    ModelRequestConfig,
)

# 1. 配置客户端连接信息（client_provider/api_base/api_key 等）
model_client_config = ModelClientConfig(
    client_provider="SiliconFlow",              # 模型提供商标识符，框架根据此值自动选择客户端实现
    api_base="https://api.siliconflow.cn/v1",   # 模型服务 URL
    api_key="sk-****************************",  # 鉴权 Token
    custom_headers={                            # 可选：为 OpenAI 兼容请求注入自定义请求头
        "Token": "tenant-token",
        "UserID": "user-a",
    },
    verify_ssl=False                            # 示例中关闭 SSL 校验，生产环境建议开启
)

# 2. 配置本次请求的模型及参数
model_request_config = ModelRequestConfig(
    model="Qwen/Qwen3-32B",     # 具体模型名称，由服务端支持的模型列表决定
    temperature=0.7,
    top_p=0.9,
)

# 3. 创建统一模型入口
model = Model(
    model_client_config=model_client_config,
    model_config=model_request_config,
)
```

> **说明**
> - 用户需要自行前往硅基流动或 OpenAI 等厂商官网注册账号，以获取可用模型的 `api_key` 和模型调用的 URL 地址 `api_base`。
> - `client_provider` 目前内置支持 `OpenAI` 和 `SiliconFlow`，框架会根据此配置自动选择对应的模型客户端实现。

### 使用 OpenAI 账户 OAuth 登录

除了通过 `api_key` 接入模型服务外，`client_provider` 还支持 `OpenAIAccount`：使用 OpenAI 账户的 OAuth 凭据接入，无需在 `ModelClientConfig` 中配置 `api_key`，框架会在调用时从本地凭据存储中解析访问令牌，并在令牌即将过期时自动刷新。

使用前需先完成一次设备码（Device Code）登录，登录成功后凭据会保存到本地凭据文件（默认路径 `~/.openjiuwen/auth.json`，可通过环境变量 `OPENJIUWEN_AUTH_FILE` 自定义）：

```python
from openjiuwen.extensions.external_provider.openai_auth import OpenAIAccountAuthManager

manager = OpenAIAccountAuthManager()


def on_device_code(device_code):
    print(f"请在浏览器中打开：{device_code.verification_uri}")
    print(f"并输入验证码：{device_code.user_code}")


manager.login_with_device_code(on_device_code=on_device_code)
```

也可以直接运行命令行脚本完成登录、查看登录状态或退出登录：

```bash
python -m openjiuwen.extensions.external_provider.openai_auth.openai_account_login login
python -m openjiuwen.extensions.external_provider.openai_auth.openai_account_login status
python -m openjiuwen.extensions.external_provider.openai_auth.openai_account_login logout
```

登录成功后，即可像使用其他厂商一样创建模型客户端：

```python
from openjiuwen.core.foundation.llm import Model, ModelClientConfig, ModelRequestConfig

model_client_config = ModelClientConfig(
    client_provider="OpenAIAccount",                     # 使用 OpenAI 账户 OAuth 免密登录，无需 api_key
    api_base="https://chatgpt.com/backend-api/codex",     # 必填：OpenAI 账户后端地址
)

model_request_config = ModelRequestConfig(
    model="gpt-5.5",
)

model = Model(
    model_client_config=model_client_config,
    model_config=model_request_config,
)
```

> **说明**
> - `ModelClientConfig` 校验时，`api_key` 仅对 `OpenAIAccount` 之外的 provider 强制要求；但 `api_base` 对所有内置 provider（含 `OpenAIAccount`）都是必填项，不会自动填充默认值，省略会在构造 `ModelClientConfig` 时直接报错。
> - 上例中的地址与 `OpenAIAccountAuthManager` / `OpenAIAccountModelCatalog` 的内置默认地址一致；`OPENJIUWEN_OPENAI_ACCOUNT_BASE_URL` 环境变量仅影响这两者在未显式传入 `base_url` 时的默认值，不会代替 `ModelClientConfig.api_base` 的必填校验。
> - 可通过 `OpenAIAccountModelCatalog.list_model_ids()` 获取当前账户可用的模型列表；若无法访问在线接口，会依次回退到本地缓存和内置模型列表。

### 配置自定义请求头

当模型服务基于 OpenAI 兼容协议且要求额外请求头时，可以通过 `custom_headers` 注入自定义请求头。例如多租户标识、业务侧用户标识、网关扩展头等。

#### 配置级请求头

可在 `ModelClientConfig` 中通过 `custom_headers` 配置默认请求头，这些头会在当前模型对象的每次调用中自动透传：

```python
from openjiuwen.core.foundation.llm import ModelClientConfig

model_client_config = ModelClientConfig(
    client_provider="OpenAI",
    api_base="https://your-openai-compatible-endpoint/v1",
    api_key="sk-****************************",
    custom_headers={
        "Token": "tenant-token",
        "UserID": "user-a",
    },
)
```

#### 请求级请求头

如果某一次请求需要临时覆盖或补充请求头，可以在 `invoke` 或 `stream` 时通过 `custom_headers` 传入：

```python
response = await model.invoke(
    messages=messages,
    custom_headers={
        "userid": "user-b",   # 会覆盖配置级的 UserID
        "X-Trace-Id": "trace-001",
    },
)
```

#### 头部合并规则

- 请求级 `custom_headers` 会覆盖配置级同名请求头。
- 请求头名称按大小写无关方式比较，例如 `UserID` 与 `userid` 视为同一个头。
- 框架会自动过滤受保护请求头：`Authorization`、`Host`、`Content-Length`、`Transfer-Encoding`、`Connection`。
- `None`、空字符串、仅空白字符的键值不会被透传。

#### 其他入口的配置方式

除直接构造 `ModelClientConfig` 外，也可以通过其他统一入口使用该能力：

```python
from openjiuwen.core.foundation.llm import init_model
from openjiuwen.core.single_agent.agents.react_agent import ReActAgentConfig

# 方式一：init_model
model = init_model(
    provider="OpenAI",
    model_name="qwen-plus",
    api_key="sk-****************************",
    api_base="https://your-openai-compatible-endpoint/v1",
    custom_headers={"Token": "tenant-token"},
)

# 方式二：ReActAgentConfig
react_agent_config = ReActAgentConfig()
react_agent_config.configure_custom_headers({
    "Token": "tenant-token",
    "UserID": "user-a",
})
```

### 准备模型输入

大模型支持以下主要输入参数，仅配置 `messages` 时，模型将基于上下文直接生成回复；同时配置 `messages` 和 `tools` 时，模型将结合上下文信息与工具定义，判断是否调用工具以完成用户请求。

- **messages**：按对话顺序排列的消息列表，每个消息对象包含发送者的角色（`role`）和消息内容（`content`），为**必选**参数。入参支持两种常见写法（`List[BaseMessage]`、`List[Dict]`）：

**写法一：使用 BaseMessage 类型**

```python
from openjiuwen.core.foundation.llm import SystemMessage, UserMessage

messages = [
    SystemMessage(content="你是一个AI助手"),
    UserMessage(content="你好")
]
```

**写法二：使用字典类型**

```python
messages = [
    {"role": "system", "content": "你是一个AI助手"},
    {"role": "user", "content": "你好"}
]
```

- **tools**：大模型可使用的工具列表，每个工具使用 JSON Schema 格式定义，详细说明所需参数、参数类型及是否必填等，为**可选**参数，仅在需要执行工具调用时使用。示例代码如下：

```python
# 工具定义
tools = [{
    "type": "function",
    "function": {
        "name": "get_weather",
        "description": "Get current weather for a location",
        "parameters": {
            "type": "object",
            "properties": {
                "location": {
                    "type": "string",
                    "description": "City and country e.g. Paris, France"
                },
                "units": {
                    "type": "string",
                    "enum": ["metric", "imperial"],
                    "description": "Temperature unit"
                }
            },
            "required": ["location"]
        }
    }
}]
```

### 调用模型

大模型的调用支持非流式 `invoke` 和流式 `stream` 两种方式，均为**异步方法**：

| 方法 | 说明 | 适用场景 |
|------|------|---------|
| `invoke` | 异步非流式调用，一次性返回完整结果 | 需要一次性拿到完整结果、对实时性要求不高的场景 |
| `stream` | 异步流式调用，逐块返回响应内容 | 需要高并发实时推送、异步非阻塞输出，或边生成边展示的场景 |

#### async invoke 方法

异步调用大模型，一次性获取完整响应结果。

**参数**：

- **messages**(Union[str, List[BaseMessage], List[dict]])：调用大模型的输入消息。必填。
- **tools**(Union[List[ToolInfo], List[dict], None], 可选)：指定模型可调用的工具列表。默认值：None。
- **temperature**(float, 可选)：控制模型输出的随机性。取值范围：[0, 1]。覆盖 ModelRequestConfig 中的配置。默认值：None（使用配置值）。
- **top_p**(float, 可选)：控制模型输出的多样性。取值范围：[0, 1]。覆盖 ModelRequestConfig 中的配置。默认值：None（使用配置值）。
- **model**(str, 可选)：指定模型名称。覆盖 ModelRequestConfig 中的配置。默认值：None（使用配置值）。
- **max_tokens**(int, 可选)：生成的最大 token 数量。默认值：None。
- **stop**(str, 可选)：停止序列。默认值：None。
- **output_parser**(BaseOutputParser, 可选)：输出解析器，用于解析模型响应内容。默认值：None。
- **timeout**(float, 可选)：本次请求的超时时间。覆盖 ModelClientConfig 中的配置。默认值：None（使用配置值）。
- **custom_headers**(Mapping[str, Any], 可选)：本次请求附加的自定义请求头。会按大小写无关方式与 `ModelClientConfig.custom_headers` 合并，并覆盖同名配置级请求头。默认值：None。

**返回**：

**AssistantMessage**，大模型返回的消息对象，包含以下主要字段：
- `role`：角色，固定为 `assistant`。
- `content`：响应内容文本。
- `tool_calls`：工具调用列表（如果有）。
- `usage_metadata`：使用统计信息。
- `finish_reason`：完成原因，`stop` 表示正常结束，`tool_calls` 表示需要调用工具。

**示例**：

```python
import asyncio
from openjiuwen.core.foundation.llm import (
    Model,
    ModelClientConfig,
    ModelRequestConfig,
    SystemMessage,
    UserMessage,
)


async def invoke_example():
    # 配置模型客户端
    model_client_config = ModelClientConfig(
        client_provider="SiliconFlow",
        api_base="https://api.siliconflow.cn/v1",
        api_key="sk-****************************",
        verify_ssl=False,
    )
    model_request_config = ModelRequestConfig(
        model="Qwen/Qwen3-32B",
        temperature=0.7,
        top_p=0.95,
    )
    model = Model(
        model_client_config=model_client_config,
        model_config=model_request_config,
    )

    # 准备模型输入数据
    messages = [
        SystemMessage(content="你是一个AI助手"),
        UserMessage(content="杭州天气")
    ]

    # 天气工具 schema 定义
    tools = [{
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "Get current weather for a location",
            "parameters": {
                "type": "object",
                "properties": {
                    "location": {
                        "type": "string",
                        "description": "City and country e.g. Paris, France"
                    },
                    "units": {
                        "type": "string",
                        "enum": ["metric", "imperial"],
                        "description": "Temperature unit"
                    }
                },
                "required": ["location"]
            }
        }
    }]

    # 调用模型
    response = await model.invoke(messages=messages, tools=tools)
    print(response)


if __name__ == "__main__":
    asyncio.run(invoke_example())
```

**输出结果**：

```
AssistantMessage(role='assistant', content='', tool_calls=[ToolCall(id='019afe192ceee268d980f8acd98ccbc1', type='function', name='get_weather', arguments='{"location": "杭州, 中国"}')], usage_metadata=UsageMetadata(model_name='Qwen/Qwen3-32B', input_tokens=156, output_tokens=23, total_tokens=179), finish_reason='tool_calls')
```

#### async stream 方法

异步流式调用大模型，逐块返回响应内容。

**参数**：

参数与 `invoke` 方法相同，包括可选的 `custom_headers`。

**返回**：

**AsyncIterator[AssistantMessageChunk]**，异步迭代器，每次迭代返回一个响应块，包含以下主要字段：
- `content`：当前块的内容增量。
- `reasoning_content`：推理内容（如果模型支持）。
- `tool_calls`：工具调用增量（如果有）。
- `usage_metadata`：使用统计信息（通常只在最后一个块中包含）。
- `finish_reason`：完成原因。

**示例**：

```python
import asyncio
from openjiuwen.core.foundation.llm import (
    Model,
    ModelClientConfig,
    ModelRequestConfig,
    SystemMessage,
    UserMessage,
)


async def stream_example():
    # 配置模型客户端
    model_client_config = ModelClientConfig(
        client_provider="SiliconFlow",
        api_base="https://api.siliconflow.cn/v1",
        api_key="sk-****************************",
        verify_ssl=False,
    )
    model_request_config = ModelRequestConfig(
        model="Qwen/Qwen3-32B",
        temperature=0.7,
        top_p=0.95,
    )
    model = Model(
        model_client_config=model_client_config,
        model_config=model_request_config,
    )

    # 准备模型输入数据
    messages = [ 
        SystemMessage(content="你是一个AI助手"),
        UserMessage(content="你好")
    ]

    # 使用 async for 遍历异步迭代器
    async for chunk in model.stream(messages=messages):
        print(chunk.content, end="", flush=True)
    print()  # 换行


if __name__ == "__main__":
    asyncio.run(stream_example())
```

**输出结果**：

```
你好
！有什么我可以
帮你的吗
？
```
