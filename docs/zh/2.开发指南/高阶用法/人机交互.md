# Human-in-the-Loop 人机交互

Human-in-the-Loop（HITL）是 openJiuwen 框架中的人机协作机制，允许 Agent 在工具执行的关键节点暂停，将决策权交还给用户。该机制基于框架的 Rail 拦截器实现，在不修改 Agent 核心执行流程的前提下，为工具调用注入"中断 → 等待用户响应 → 恢复执行"的交互能力。

典型的应用场景包括：

- **敏感操作确认**：在文件删除、资金转账等高风险工具执行前，暂停并请求用户批准或拒绝。
- **信息补全**：Agent 在执行过程中发现缺少必要信息时，主动向用户提问并等待回答。
- **执行过程审计**：为关键操作建立人工审批节点，确保 Agent 的自主行为始终处于人类监督之下。

***

## 1. 内置 Rail

HITL 机制基于 Rail 拦截器实现。框架提供两种内置 Rail，通过 `register_rail` 注册到 Agent 上。

### 1.1 ConfirmInterruptRail —— 操作确认

拦截指定工具的调用，暂停执行并请求用户批准或拒绝。

**配置方式**：

通过 `tool_names` 参数指定需要拦截的工具，当 Agent 调用这些工具时会被暂停：

```python
from openjiuwen.harness.rails import ConfirmInterruptRail

rail = ConfirmInterruptRail(tool_names=["read", "write", "delete"])
await agent.register_rail(rail)
```

**响应字段**：

用户通过 `InteractiveInput` 提供决策，包含以下字段：

- `approved`：是否批准执行（必填，True 表示批准，False 表示拒绝）
- `feedback`：用户反馈信息（可选）
- `auto_confirm`：是否自动确认后续同名工具（可选，默认 False）

**批准执行**：

```python
interactive_input = InteractiveInput()
interactive_input.update(tool_call_id, {
    "approved": True,
    "feedback": "确认执行"
})
```

**拒绝执行**：

拒绝后工具不会执行，反馈信息会作为 `ToolMessage` 返回给 Agent，Agent 可以根据反馈调整后续行为：

```python
interactive_input = InteractiveInput()
interactive_input.update(tool_call_id, {
    "approved": False,
    "feedback": "此操作太危险，拒绝执行"
})
```

完整示例见 [5.1 工具执行确认](#51-工具执行确认)。

### 1.2 AskUserRail —— 用户输入收集

让 Agent 主动向用户询问信息，适用于信息补全场景。

**配置方式**：

注册后 Agent 可以通过内置的 `ask_user` 工具来收集用户输入：

```python
from openjiuwen.harness.rails import AskUserRail

rail = AskUserRail()
await agent.register_rail(rail)
```

**多问题模式**：

`ask_user` 工具支持多问题模式，可一次性向用户提出1-4个问题，每个问题可带2-4个选项。中断返回后，通过 `state` 字段中的 `questions` 获取问题列表：

```python
state_list = result.get("state", [])
for state in state_list:
    payload = state.payload
    tool_call_id = payload.id
    interrupt_value = payload.value
    
    questions = interrupt_value.questions  # 问题列表
    for q in questions:
        print(f"问题: {q['question']}")
        print(f"选项: {q.get('options', [])}")
```

**响应字段**：

用户通过 `InteractiveInput` 提供答案：

- `answers`：问题文本到答案的映射字典（必填），键为问题的完整文本（`question` 字段值），值为用户的答案

**返回答案**：

```python
# 从 questions 中获取问题完整文本
first_question = questions[0]['question']

interactive_input = InteractiveInput()
interactive_input.update(tool_call_id, {"answers": {first_question: "user_input.txt"}})
```

完整示例见 [5.2 用户输入收集](#52-用户输入收集)。

## 2. 处理中断

无论使用哪种 Rail，处理中断的流程都是相同的：检测中断、解析信息、构造响应、恢复执行。

### 2.1 检测中断

当工具被 Rail 拦截时，`Runner.run_agent` 返回的结果中 `result_type` 会标记为 `"interrupt"`：

```python
if result.get("result_type") == "interrupt":
    interrupt_ids = result.get("interrupt_ids", [])
```

### 2.2 解析中断信息

中断结果的完整结构如下：
```python
{
    "result_type": "interrupt",
    "interrupt_ids": ["call_dd26eaa14529440c81b54eab"],
    "state": [
        {
            "type": "__interaction__",
            "index": 0,
            "payload": {
                "id": "call_dd26eaa14529440c81b54eab",
                "value": {
                    "message": "Please approve or reject?",
                    "payload_schema": {
                        "description": "Payload for user confirmation response.",
                        "properties": {
                            "approved": {"title": "Approved", "type": "boolean"},
                            "feedback": {"default": "", "title": "Feedback", "type": "string"},
                            "auto_confirm": {"default": False, "title": "Auto Confirm", "type": "boolean"}
                        },
                        "required": ["approved"],
                        "title": "ConfirmPayload",
                        "type": "object"
                    },
                    "auto_confirm_key": "read",
                    "tool_name": "read",
                    "tool_call_id": "call_dd26eaa14529440c81b54eab",
                    "tool_args": '{"filepath": "/tmp/test1.txt"}',
                    "index": 0
                }
            }
        }
    ]
}
```

**字段说明**：
- `result_type`：结果类型，`"interrupt"` 表示工具被拦截
- `interrupt_ids`：被拦截的工具调用 ID 列表
- `state`：中断信息列表，每个元素包含一个中断的详细信息

**Payload 结构**：
每个中断的 `payload` 包含以下字段：
- `id`：工具调用 ID，用于恢复执行时标识具体的中断
- `value`：中断的详细信息，包含以下字段：
  - `message`：提示消息
  - `payload_schema`：定义用户响应格式的 JSON Schema
  - `auto_confirm_key`：自动确认的 key，用于 auto_confirm 功能
  - `tool_name`：工具名称
  - `tool_call_id`：工具调用 ID
  - `tool_args`：工具参数
  - `index`：工具调用索引

从 `state` 字段中获取中断的详细信息：

```python
state_list = result.get("state", [])
for state in state_list:
    payload = state.payload
    tool_call_id = payload.id
    interrupt_value = payload.value
    
    tool_name = interrupt_value.tool_name
    tool_args = interrupt_value.tool_args
    message = interrupt_value.message
    payload_schema = interrupt_value.payload_schema
```

### 2.3 构造响应并恢复执行

用户通过 `InteractiveInput` 传递决策，再次调用 `Runner.run_agent` 恢复执行。响应格式应该根据 `payload_schema` 来填写：

```python
interactive_input = InteractiveInput()
interactive_input.update(tool_call_id, {
    "approved": True,
    "feedback": "确认执行",
    "auto_confirm": False
})

result = await Runner.run_agent(
    agent=agent,
    inputs={"query": interactive_input, "conversation_id": "session_1"},
)
```

不同 Rail 的 `payload_schema` 可能不同，用户需要根据实际的 schema 来填写 `InteractiveInput`。

### 2.4 Session 管理

自动确认配置存储在 Session 中，如果 Session 配置了持久化，自动确认配置也会随之持久化。

```python
await agent.clear_session(session_id)
```

清除 Session 后，自动确认配置会被清除，后续调用会重新触发中断。

## 3. 进阶场景

### 3.1 自动确认

用户确认时设置 `auto_confirm=True`，后续同名工具调用将自动通过，无需再次确认：

```python
interactive_input = InteractiveInput()
interactive_input.update(tool_call_id, {
    "approved": True,
    "feedback": "确认，后续自动通过",
    "auto_confirm": True
})

result2 = await Runner.run_agent(
    agent=agent,
    inputs={"query": interactive_input, "conversation_id": "session_1"},
)

result3 = await Runner.run_agent(
    agent=agent,
    inputs={"query": "再次读取 /tmp/other.txt", "conversation_id": "session_1"},
)

print(f"结果类型: {result3.get('result_type')}")
```

### 3.2 并发工具中断

当多个工具并发调用时，所有中断会批量返回，用户需要为每个中断提供决策：

```python
result1 = await Runner.run_agent(
    agent=agent,
    inputs={"query": "同时读取 a.txt 和 b.txt", "conversation_id": "session_1"},
)

interrupt_ids = result1.get("interrupt_ids", [])

# 两个工具都需要确认
assert len(interrupt_ids) == 2

# 按顺序提供决策
interactive_input = InteractiveInput()
interactive_input.update(interrupt_ids[0], {"approved": True, "feedback": "确认"})
interactive_input.update(interrupt_ids[1], {"approved": True, "feedback": "确认"})

result2 = await Runner.run_agent(
    agent=agent,
    inputs={"query": interactive_input, "conversation_id": "session_1"},
)
```

每个工具调用都有唯一的 `tool_call_id`，可以通过 `interrupt_ids` 获取。同时，`state` 字段包含每个工具调用的详细信息，可用于区分不同的工具调用。

### 3.3 流式输出中的中断

在流式输出场景中，中断会在流式输出中检测到，需要再次调用流式输出来恢复执行：

```python
# 第一次流式调用：检测中断
outputs1 = []
interrupt_detected = False
tool_call_id = None

async for output in Runner.run_agent_streaming(
    agent=agent,
    inputs={"query": "读取 /tmp/test.txt", "conversation_id": "session_1"},
):
    outputs1.append(output)
    if output.type == '__interaction__':
        interrupt_detected = True
        tool_call_id = output.payload.id

# 构造用户响应
interactive_input = InteractiveInput()
interactive_input.update(tool_call_id, {
    "approved": True,
    "feedback": "确认执行",
    "auto_confirm": True
})

# 第二次流式调用：恢复执行
outputs2 = []
async for output in Runner.run_agent_streaming(
    agent=agent,
    inputs={"query": interactive_input, "conversation_id": "session_1"},
):
    outputs2.append(output)
```

### 3.4 子Agent嵌套中断

当使用 AgentAsTool 模式时，子Agent内部的中断会自动向上传播到父Agent，用户只需在最外层进行确认。恢复时，确认信息会自动传递到子Agent。

**中断传播流程**：

```
main_agent
  └─ 调用 sub_agent
        └─ 调用 read 工具 → [Rail 拦截] → 中断向上传播
              ↓
        返回中断结果到 main_agent
              ↓
        用户确认（只需在最外层操作）
              ↓
        确认信息向下传递到 sub_agent
              ↓
        恢复执行
```

**Auto Confirm 的传递**：

自动确认配置统一存储在主Agent的 Session 中，会自动传递给子Agent。当子Agent调用工具时，会从父Agent的 Session 中读取 auto_confirm 配置。

子Agent的中断对用户是透明的，用户只需在最外层Agent进行确认即可。auto_confirm 配置会自动传递到所有子Agent。

完整示例见 [5.3 子Agent嵌套中断](#53-子agent嵌套中断)。

## 4. 自定义中断 Rail

开发者可以通过继承 `BaseInterruptRail` 实现自定义中断逻辑。

### 4.1 三种决策类型

自定义 Rail 需要实现 `resolve_interrupt` 方法，返回以下三种决策之一：

| 决策类型            | 说明   | 使用场景           |
| --------------- | ---- | -------------- |
| ApproveResult   | 批准执行 | 用户确认执行，或需要修改参数 |
| RejectResult    | 拒绝执行 | 用户拒绝执行，或验证失败   |
| InterruptResult | 中断等待 | 首次拦截，等待用户输入    |

### 4.2 扩展示例

以下示例展示了两个主要的扩展点：自定义决策引擎和细粒度自动确认。

```python
from typing import Any, Optional
from openjiuwen.core.foundation.llm.schema.tool_call import ToolCall
from openjiuwen.core.single_agent.interrupt.response import InterruptRequest
from openjiuwen.core.single_agent.rail.base import AgentCallbackContext
from openjiuwen.harness.rails import ConfirmInterruptRail, ConfirmPayload
from openjiuwen.harness.rails.interrupt.decision import InterruptDecision

class BashPermissionRail(ConfirmInterruptRail):
    """Bash 权限控制 Rail - 展示自定义决策引擎和细粒度自动确认
    
    功能：
    1. 自定义决策引擎：检查命令是否包含危险模式（如 rm *）
    2. 细粒度自动确认：基于命令内容生成不同的 auto_confirm key
    """
    
    def __init__(self, tool_names):
        super().__init__(tool_names=tool_names)
    
    async def resolve_interrupt(
        self,
        ctx: AgentCallbackContext,
        tool_call: Optional[ToolCall],
        user_input: Optional[Any],
        auto_confirm_config: Optional[dict] = None,
    ) -> InterruptDecision:
        """自定义决策引擎：控制三种决策逻辑"""
        tool_name = tool_call.name if tool_call else ""
        command = self._get_command(tool_call)
        auto_confirm_key = self._get_auto_confirm_key(command)
        
        # 检查是否已经自动确认（首次调用时检查）
        if user_input is None:
            if self._is_auto_confirmed(auto_confirm_config, auto_confirm_key):
                return self.approve()
            
            # 检查是否包含危险模式
            if "rm *" in command or "rm -rf /" in command:
                return self.reject(tool_result="命令包含危险操作，被拒绝执行")
            
            return self.interrupt(InterruptRequest(
                message=f"执行 bash 命令需要授权: {command}",
                payload_schema=ConfirmPayload.to_schema(),
                auto_confirm_key=auto_confirm_key,
            ))
        
        # 用户响应后处理
        if user_input.get("approved"):
            return self.approve()
        else:
            return self.reject(tool_result=user_input.get("feedback", "用户拒绝"))
    
    def _get_auto_confirm_key(self, command: str) -> str:
        """细粒度自动确认：基于命令内容生成 key"""
        # 简单示例：基于命令的第一个词生成 key
        # ls /tmp -> bash_ls
        # cat file.txt -> bash_cat
        cmd_parts = command.strip().split()
        if cmd_parts:
            return f"bash_{cmd_parts[0]}"
        return "bash"
    
    def _get_command(self, tool_call):
        """获取命令内容"""
        if tool_call.name == "bash":
            args = json.loads(tool_call.arguments) if isinstance(tool_call.arguments, str) else tool_call.arguments
            return args.get("command", "")
        return ""
```

**使用示例**：

```python
rail = BashPermissionRail(tool_names=["bash"])
await agent.register_rail(rail)

# 场景1：自定义决策引擎
# 执行 "rm *" -> 直接拒绝（包含危险模式）

# 场景2：细粒度自动确认
# 执行 "ls /tmp" -> 中断 -> 确认（auto_confirm=True, key: bash_ls）
# 再次执行 "ls /home" -> 自动通过（key: bash_ls 匹配）
# 执行 "cat file.txt" -> 中断（key: bash_cat 不匹配）
```

## 5. 完整示例

以下示例使用 `@tool` 注解定义工具，展示 HITL 的完整使用流程。

### 5.1 工具执行确认

以下示例展示了工具执行确认流程：读取文件时请求用户确认。

```python
import asyncio
import json
from typing import Annotated

from pydantic import Field

from openjiuwen.core.runner import Runner
from openjiuwen.core.session import InteractiveInput
from openjiuwen.core.single_agent.agents.react_agent import ReActAgent, ReActAgentConfig
from openjiuwen.core.single_agent.schema.agent_card import AgentCard
from openjiuwen.harness.rails import ConfirmInterruptRail
from openjiuwen.core.foundation.tool import tool


@tool
async def read_file(
    file_path: Annotated[str, Field(description="要读取的文件路径")]
) -> str:
    """读取文件内容"""
    return "Hello"


async def main():
    await Runner.start()

    try:
        agent = ReActAgent(card=AgentCard(id="confirm_agent"))
        config = ReActAgentConfig()
        config.configure_model_client(
            provider="openai",
            api_key="your-api-key",
            api_base="https://api.openai.com/v1",
            model_name="gpt-4",
        )
        agent.configure(config)

        Runner.resource_mgr.add_tool(read_file)
        agent.ability_manager.add(read_file.card)

        rail = ConfirmInterruptRail(tool_names=["read_file"])
        await agent.register_rail(rail)

        print("=== 第一次调用：触发中断 ===")
        result1 = await Runner.run_agent(
            agent=agent,
            inputs={"query": "请读取 /tmp/test.txt 文件", "conversation_id": "session_1"},
        )

        print(f"结果类型: {result1.get('result_type')}")

        if result1.get("result_type") == "interrupt":
            interrupt_ids = result1.get("interrupt_ids", [])
            print(f"中断数量: {len(interrupt_ids)}")

            state_list = result1.get("state", [])
            for state in state_list:
                payload = state.payload
                tool_call_id = payload.id
                interrupt_value = payload.value

                print(f"\n中断详情:")
                print(f"  工具名称: {interrupt_value.tool_name}")
                print(f"  工具参数: {interrupt_value.tool_args}")
                print(f"  提示消息: {interrupt_value.message}")
                print(f"  自动确认key: {interrupt_value.auto_confirm_key}")

                interactive_input = InteractiveInput()
                interactive_input.update(tool_call_id, {
                    "approved": True,
                    "feedback": "确认读取"
                })

                print("\n=== 第二次调用：恢复执行 ===")
                result2 = await Runner.run_agent(
                    agent=agent,
                    inputs={"query": interactive_input, "conversation_id": "session_1"},
                )

                print(f"结果类型: {result2.get('result_type')}")
    finally:
        await Runner.stop()


if __name__ == "__main__":
    asyncio.run(main())
```

**输出示例**：

```
=== 第一次调用：触发中断 ===
结果类型: interrupt
中断数量: 1

中断详情:
  工具名称: read_file
  工具参数: {"file_path": "/tmp/test.txt"}
  提示消息: Please approve or reject?
  自动确认 key: read_file

=== 第二次调用：恢复执行 ===
结果类型: answer
```

### 5.2 用户输入收集

以下示例展示了 Agent 主动向用户询问信息：

```python
import asyncio
import json

from openjiuwen.core.runner import Runner
from openjiuwen.core.session import InteractiveInput
from openjiuwen.core.single_agent.agents.react_agent import ReActAgent, ReActAgentConfig
from openjiuwen.core.single_agent.schema.agent_card import AgentCard
from openjiuwen.harness.rails import AskUserRail


async def main():
    await Runner.start()

    try:
        agent = ReActAgent(card=AgentCard(id="ask_user_agent"))
        config = ReActAgentConfig()
        config.configure_model_client(
            provider="openai",
            api_key="your-api-key",
            api_base="https://api.openai.com/v1",
            model_name="gpt-4",
        )
        config.configure_prompt_template([
            {"role": "system", "content": ""}
        ])
        agent.configure(config)

        rail = AskUserRail()
        await agent.register_rail(rail)

        print("=== 第一次调用：触发中断 ===")
        result1 = await Runner.run_agent(
            agent=agent,
            inputs={"query": "Please ask the user for the filename they want. use ask_user tool",
                    "conversation_id": "session_1"},
        )

        print(f"结果类型: {result1.get('result_type')}")

        if result1.get("result_type") == "interrupt":
            interrupt_ids = result1.get("interrupt_ids", [])
            print(f"中断数量: {len(interrupt_ids)}")

            state_list = result1.get("state", [])
            for state in state_list:
                payload = state.payload
                tool_call_id = payload.id
                interrupt_value = payload.value

                print(f"\n中断详情:")
                print(f"  问题列表: {json.dumps(interrupt_value.questions, indent=4, ensure_ascii=False)}")
                print(f"  payload_schema: {json.dumps(interrupt_value.payload_schema, indent=4, ensure_ascii=False)}")

                # 从 questions 中获取问题完整文本
                first_question = interrupt_value.questions[0]['question']
                print(f"\n用户回答: my_document.txt")
                interactive_input = InteractiveInput()
                interactive_input.update(tool_call_id, {"answers": {first_question: "my_document.txt"}})

                print("\n=== 第二次调用：恢复执行 ===")
                result2 = await Runner.run_agent(
                    agent=agent,
                    inputs={"query": interactive_input, "conversation_id": "session_1"},
                )

                print(f"结果类型: {result2.get('result_type')}")
    finally:
        await Runner.stop()


if __name__ == "__main__":
    asyncio.run(main())
```

            {"label": "file1.txt", "description": "默认文件"}

        "multi_select": false

]
  payload_schema: {
    "description": "Payload for user input response.",
    "properties": {
        "answers": {
            "additionalProperties": {"type": "string"},
            "description": "Question text to answer mapping",

### 5.3 子Agent嵌套中断

以下示例展示了子Agent嵌套场景下的中断传播和自动确认：

```python
import asyncio
from typing import Annotated

from pydantic import Field

from openjiuwen.core.runner import Runner
from openjiuwen.core.session import InteractiveInput
from openjiuwen.core.single_agent.agents.react_agent import ReActAgent, ReActAgentConfig
from openjiuwen.core.single_agent.schema.agent_card import AgentCard
from openjiuwen.harness.rails import ConfirmInterruptRail
from openjiuwen.core.foundation.tool import tool


@tool
async def read_file(
    file_path: Annotated[str, Field(description="要读取的文件路径")]
) -> str:
    """读取文件内容"""
    return "Hello"


async def main():
    await Runner.start()

    try:
        Runner.resource_mgr.add_tool(read_file)

        print("=== 创建子Agent ===")
        sub_agent_card = AgentCard(
            id="sub_agent",
            name="sub_agent",
            description="一个可以读取文件的子Agent",
            input_params={
                "type": "object",
                "properties": {
                    "query": {"description": "任务描述", "type": "string"},
                },
                "required": ["query"],
            },
        )
        sub_agent = ReActAgent(card=sub_agent_card)
        sub_config = ReActAgentConfig()
        sub_config.configure_model_client(
            provider="openai",
            api_key="your-api-key",
            api_base="https://api.openai.com/v1",
            model_name="gpt-4",
        )
        sub_agent.configure(sub_config)

        sub_agent.ability_manager.add(read_file.card)

        sub_rail = ConfirmInterruptRail(tool_names=["read_file"])
        await sub_agent.register_rail(sub_rail)
        print("子Agent已注册 read_file 中断 Rail")

        print("\n=== 创建主Agent ===")
        main_agent = ReActAgent(card=AgentCard(id="main_agent"))
        main_config = ReActAgentConfig()
        main_config.configure_model_client(
            provider="openai",
            api_key="your-api-key",
            api_base="https://api.openai.com/v1",
            model_name="gpt-4",
        )
        main_agent.configure(main_config)

        Runner.resource_mgr.add_agent(sub_agent.card, agent=lambda: sub_agent)
        main_agent.ability_manager.add(sub_agent.card)
        print("主Agent已添加子Agent作为工具")

        print("\n=== 第一次调用：触发中断 ===")
        result1 = await Runner.run_agent(
            agent=main_agent,
            inputs={"query": "调用子Agent读取文件 /tmp/test.txt", "conversation_id": "session_1"},
        )

        print(f"结果类型: {result1.get('result_type')}")

        if result1.get("result_type") == "interrupt":
            interrupt_ids = result1.get("interrupt_ids", [])
            print(f"中断数量: {len(interrupt_ids)}")

            state_list = result1.get("state", [])
            for state in state_list:
                payload = state.payload
                tool_call_id = payload.id
                interrupt_value = payload.value

                print(f"\n中断详情:")
                print(f"  工具名称: {interrupt_value.tool_name}")
                print(f"  提示消息: {interrupt_value.message}")
                print(f"  注意：中断来自子Agent，但在主Agent处统一处理")

                interactive_input = InteractiveInput()
                interactive_input.update(tool_call_id, {
                    "approved": True,
                    "feedback": "确认",
                    "auto_confirm": True
                })

                print("\n=== 第二次调用：恢复执行 ===")
                result2 = await Runner.run_agent(
                    agent=main_agent,
                    inputs={"query": interactive_input, "conversation_id": "session_1"},
                )

                print(f"结果类型: {result2.get('result_type')}")

                if result2.get("result_type") == "answer":
                    messages = result2.get("messages", [])
                    if messages:
                        print(f"最终回复: {messages[-1].content[:100]}")

                print("\n=== 第三次调用：测试自动确认 ===")
                result3 = await Runner.run_agent(
                    agent=main_agent,
                    inputs={"query": "再次调用子Agent读取文件 /tmp/test.txt", "conversation_id": "session_1"},
                )

                print(f"结果类型: {result3.get('result_type')}")
                print("注意：由于启用了 auto_confirm，这次调用没有触发中断")
    finally:
        await Runner.stop()


if __name__ == "__main__":
    asyncio.run(main())
```
