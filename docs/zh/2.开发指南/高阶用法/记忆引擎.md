# 记忆引擎（LongTermMemory）

记忆引擎（`LongTermMemory`）是 openJiuwen 在当前版本中提供的统一记忆管理组件，负责管理用户的对话消息、变量记忆和长期用户画像。

和旧版文档中的 `MemoryEngine/SysMemConfig/MemoryConfig` 不同，**0.1.4 起记忆能力完全由 `LongTermMemory + MemoryEngineConfig + MemoryScopeConfig + AgentMemoryConfig` 这套接口负责**，并以 `scope_id` 区分不同的业务/Agent 场景。

本章节将严格基于源码 `openjiuwen.core.memory` 下的对外接口进行说明。

## 核心概念与配置类

记忆引擎的核心类型定义在 `openjiuwen.core.memory` 与 `openjiuwen.core.memory.config` 模块中：

- **`LongTermMemory`**（记忆引擎本体）  
  - 负责注册底层存储（KV Store / Vector Store / DB）、设置全局引擎配置、管理各个 `scope` 的记忆配置以及读写用户记忆。

- **`MemoryEngineConfig`**（全局引擎配置）  
  定义引擎级别的通用配置：
  - `default_model_cfg: ModelRequestConfig`：默认用于生成记忆的大模型请求参数（模型名、温度等）。
  - `default_model_client_cfg: ModelClientConfig`：默认的大模型客户端配置（`client_provider/api_base/api_key/verify_ssl` 等）。
  - `forbidden_variables: str`：禁止记忆的变量（逗号分隔的变量名）。默认值：`""`（不禁止任何变量）。
  - `input_msg_max_len: int`：输入消息最大长度（默认 8192）。
  - `crypto_key: bytes`：用于加密存储中敏感字段的 AES 密钥（长度必须为 32 字节；为空表示不加密）。

- **`MemoryScopeConfig`**（作用域级配置）  
  用于为不同 `scope_id` 定义独立的模型 / 向量配置：
  - `model_cfg: ModelRequestConfig`：该 scope 下使用的大模型请求配置。
  - `model_client_cfg: ModelClientConfig`：该 scope 下使用的大模型客户端配置。
  - `embedding_cfg: EmbeddingConfig`：该 scope 下使用的嵌入模型配置（`model_name/base_url/api_key`）。

- **`AgentMemoryConfig`**（Agent 级记忆策略配置）  
  描述某个智能体希望提取和管理哪些"变量记忆"、"长期记忆"、"用户画像记忆"、"用户摘要记忆"：
  - `mem_variables: list[Param]`：变量记忆配置列表（每个 `Param` 定义一个变量名、说明、类型、是否必填等）。
  - `enable_long_term_mem: bool`：是否开启长期记忆（默认 `True`）。
  - `enable_user_profile: bool`：是否开启用户画像（默认 `True`）。
  - `enable_semantic_memory: bool`：是否开启语义记忆（默认 `True`）。
  - `enable_episodic_memory: bool`：是否开启情景记忆（默认 `True`）。
  - `enable_summary_memory: bool`：是否开启用户摘要记忆（默认 `True`）。

> 这些类在源码中分别定义于：  
> `openjiuwen.core.memory.config.config` 与 `openjiuwen.core.memory.__init__`，可以通过 `from openjiuwen.core.memory import MemoryEngineConfig, MemoryScopeConfig, AgentMemoryConfig, LongTermMemory` 直接导入。

## 配置类中大模型和嵌入模型配置详解

记忆引擎依赖三个核心配置类来定义大模型和嵌入模型的参数。这些配置类定义在 `openjiuwen.core.foundation.llm.schema.config` 和 `openjiuwen.core.foundation.store.base_embedding` 模块中。

### ModelRequestConfig（大模型请求配置）

`ModelRequestConfig` 用于配置大模型请求参数，控制模型的生成行为。

| 参数名                    | 类型          | 必填 | 默认值    | 说明                              |
| ---------------------- | ----------- | -- | ------ |---------------------------------|
| `model` / `model_name` | str         | 否  | `""`   | 模型名称，如 `gpt-4`。支持别名 `model`     |
| `temperature`          | float       | 否  | `0.95` | 温度参数，控制输出随机性。值越大输出越随机，值越小输出越确定,取值范围为[0, 1] |
| `top_p`                | float       | 否  | `0.1`  | Top-p 采样参数，控制核采样范围，取值范围为[0, 1]              |
| `max_tokens`           | int \| None | 否  | `None` | 最大生成 token 数量                   |
| `stop`                 | str \| None | 否  | `None` | 停止序列，模型遇到该序列时停止生成               |

**使用示例：**

```python
from openjiuwen.core.foundation.llm.schema.config import ModelRequestConfig

model_cfg = ModelRequestConfig(
    model="gpt-4",
    temperature=0.0
)
```

### ModelClientConfig（大模型客户端配置）

`ModelClientConfig` 用于配置大模型客户端连接参数，包括 API 密钥、服务地址等。

| 参数名               | 类型                  | 必填 | 默认值    | 说明                                                                  |
| ----------------- | ------------------- | - | ------ | ------------------------------------------------------------------- |
| `client_id`       | str                 | 否 | UUID   | 客户端唯一标识符，用于在 Runner 中注册                                             |
| `client_provider` | ProviderType \| str | 是 | -      | 服务提供商标识，支持枚举值：`OpenAI`、`OpenRouter`、`SiliconFlow`、`DashScope`|
| `api_key`         | str                 | 是 | -      | API 密钥                                                              |
| `api_base`        | str                 | 是 | -      | API 基础 URL 地址                                                       |
| `timeout`         | float               | 否 | `60.0` | 请求超时时间（秒）                                                |
| `max_retries`     | int                 | 否 | `3`    | 最大重试次数                                                              |
| `verify_ssl`      | bool                | 否 | `True` | 是否验证 SSL 证书                                                         |
| `ssl_cert`        | str \| None         | 否 | `None` | SSL 证书文件路径                                                          |

**支持的 ProviderType 枚举值：**

```python
class ProviderType(str, Enum):
    OpenAI = "OpenAI"           # OpenAI 官方 API
    OpenRouter = "OpenRouter"   # OpenRouter 聚合服务
    SiliconFlow = "SiliconFlow" # SiliconFlow 平台
    DashScope = "DashScope"     # 阿里云灵积平台
```

**使用示例：**

```python
from openjiuwen.core.foundation.llm.schema.config import ModelClientConfig, ProviderType

client_cfg = ModelClientConfig(
    client_id="my_memory_client",
    client_provider=ProviderType.OpenAI,      
    api_key="sk-xxxxxxxx",
    api_base="https://api.openai.com/v1",
    timeout=120.0,                             
    max_retries=3,
    verify_ssl=False,
)
```

**参数限制说明：**

- `client_provider`：必须是已注册的服务提供商，否则会抛出 `MODEL_PROVIDER_INVALID` 错误
- `api_key` 和 `api_base`：必填参数，不能为空

### EmbeddingConfig（嵌入模型配置）

`EmbeddingConfig` 用于配置嵌入模型参数，用于向量检索和语义搜索。

| 参数名          | 类型          | 必填 | 默认值    | 说明                |
| ------------ | ----------- | - | ------ |-------------------|
| `model_name` | str         | 是 | -      | 嵌入模型名称            |
| `base_url`   | str         | 是 | -      | API 基础 URL 地址     |
| `api_key`    | str \| None | 否 | `None` | API 密钥（部分服务可能不需要） |

**使用示例：**

```python
from openjiuwen.core.foundation.store.base_embedding import EmbeddingConfig

embedding_cfg = EmbeddingConfig(
    model_name="text-embedding-3",       # 嵌入模型名称
    base_url="https://api.openai.com/v1/embeddings",      # API 地址
    api_key="sk-xxxxxxxx",                     # API 密钥
)
```

### 配置类关系图

```
MemoryEngineConfig（全局引擎配置）
├── default_model_cfg: ModelRequestConfig      # 默认模型请求配置
└── default_model_client_cfg: ModelClientConfig # 默认模型客户端配置

MemoryScopeConfig（作用域配置）
├── model_cfg: ModelRequestConfig              # 作用域模型请求配置
├── model_client_cfg: ModelClientConfig        # 作用域模型客户端配置
└── embedding_cfg: EmbeddingConfig             # 作用域嵌入模型配置
```

> **配置优先级**：当同时设置了全局配置和作用域配置时，作用域配置优先。即 `MemoryScopeConfig` 中的配置会覆盖 `MemoryEngineConfig` 中的默认配置。

## 创建记忆引擎实例

下面示例演示如何基于 `LongTermMemory` 创建一个完整可用的记忆引擎，包括：

1. 注册 KV 存储、向量存储、关系型数据库存储；
2. 配置全局引擎参数；
3. 创建并返回可复用的 `LongTermMemory` 实例。

```python
import os
import asyncio
from sqlalchemy.ext.asyncio import create_async_engine

from openjiuwen.core.memory import (
    LongTermMemory,
    MemoryEngineConfig,
    MemoryScopeConfig,
    AgentMemoryConfig,
)
from openjiuwen.core.foundation.store import create_vector_store
from openjiuwen.core.foundation.store.kv.in_memory_kv_store import InMemoryKVStore
from openjiuwen.core.foundation.store.db.default_db_store import DefaultDbStore
from openjiuwen.core.foundation.llm.schema.config import ModelClientConfig, ModelRequestConfig
from openjiuwen.core.common.schema.param import Param
from openjiuwen.core.retrieval.common.config import EmbeddingConfig
from openjiuwen.core.foundation.llm.schema.message import BaseMessage


async def create_memory_engine() -> LongTermMemory:
    """初始化并返回 LongTermMemory 实例"""
    engine = LongTermMemory()

    # 1. 创建底层存储
    kv_store = InMemoryKVStore()

    # 向量存储（使用 create_vector_store 创建）
    # Milvus 示例
    vector_store = create_vector_store(
        "milvus",
        milvus_uri=os.getenv("MILVUS_URI", f"http://{os.getenv('MILVUS_HOST', 'localhost')}:{os.getenv('MILVUS_PORT', '19530')}"),
        milvus_token=os.getenv("MILVUS_TOKEN"),
    )

    # 关系型数据库存储（基于 SQLAlchemy AsyncEngine + DefaultDbStore）
    db_user = os.getenv("DB_USER", "user")
    db_password = os.getenv("DB_PASSWORD", "password")
    db_host = os.getenv("DB_HOST", "127.0.0.1")
    db_port = os.getenv("DB_PORT", "3306")
    agent_db_name = os.getenv("AGENT_DB_NAME", "agent_memory")

    async_engine = create_async_engine(
        f"mysql+aiomysql://{db_user}:{db_password}@{db_host}:{db_port}/{agent_db_name}?charset=utf8mb4",
        pool_size=20,
        max_overflow=20,
    )
    db_store = DefaultDbStore(async_engine)

    # 注册存储
    await engine.register_store(
        kv_store=kv_store,
        vector_store=vector_store,
        db_store=db_store,
    )

    # 2. 设置全局引擎配置（默认大模型 + 加密配置）
    default_model_cfg = ModelRequestConfig(
        model=os.getenv("MEMORY_MODEL_NAME", "<model_name>"),
        temperature=0.0,
    )
    default_model_client_cfg = ModelClientConfig(
        client_id="default_memory_llm",
        client_provider=os.getenv("MEMORY_MODEL_PROVIDER", "OpenAI"),
        api_key=os.getenv("MEMORY_MODEL_API_KEY", "sk-xxxx"),
        api_base=os.getenv("MEMORY_MODEL_API_BASE", "https://api.openai.com/v1"),
        verify_ssl=False,
    )

    # crypto_key 必须是 32 字节；为空则不启用加密
    crypto_key_env = os.getenv("SERVER_AES_MASTER_KEY_ENV", "")
    crypto_key = crypto_key_env.encode("utf-8")[:32].ljust(32, b"\0") if crypto_key_env else b""

    engine_config = MemoryEngineConfig(
        default_model_cfg=default_model_cfg,
        default_model_client_cfg=default_model_client_cfg,
        crypto_key=crypto_key,
    )
    engine.set_config(engine_config)

    return engine
```

> 注意：与旧版 `MemoryEngine.register_store(...).create_mem_engine_instance(SysMemConfig)` 不同，`LongTermMemory` 的初始化分为两步：  
> 1）`await engine.register_store(...)` 注册存储；  
> 2）`engine.set_config(MemoryEngineConfig(...))` 设置全局配置。


## 配置作用域（scope）与 Agent 记忆策略

记忆引擎通过 `scope_id` 将不同业务/Agent 的记忆隔离开来。每个 `scope_id` 对应一套 `MemoryScopeConfig` 和若干 `AgentMemoryConfig`。

### 配置 MemoryScopeConfig（作用域级模型与向量配置）

```python
from openjiuwen.core.retrieval.common.config import EmbeddingConfig
from openjiuwen.core.memory import MemoryScopeConfig


async def configure_scope(engine: LongTermMemory) -> None:
    scope_id = "app_demo_scope"

    scope_model_cfg = ModelRequestConfig(
        model=os.getenv("MEMORY_SCOPE_MODEL", "<model_name>"),
        temperature=0.1,
    )
    scope_model_client_cfg = ModelClientConfig(
        client_id="scope_llm_client",
        client_provider=os.getenv("MEMORY_SCOPE_PROVIDER", "OpenAI"),
        api_key=os.getenv("MEMORY_SCOPE_API_KEY", "sk-xxxx"),
        api_base=os.getenv("MEMORY_SCOPE_API_BASE", "https://api.openai.com/v1"),
        verify_ssl=False,
    )
    embed_cfg = EmbeddingConfig(
        model_name=os.getenv("EMBED_MODEL_NAME", "text-embedding-v3"),
        api_key=os.getenv("EMBED_API_KEY", "sk-embed-xxx"),
        base_url=os.getenv("EMBED_API_BASE", "https://api.openai.com/v1/embeddings"),
    )

    scope_cfg = MemoryScopeConfig(
        model_cfg=scope_model_cfg,
        model_client_cfg=scope_model_client_cfg,
        embedding_cfg=embed_cfg,
    )

    ok = await engine.set_scope_config(scope_id, scope_cfg)
    assert ok, "设置 MemoryScopeConfig 失败"
```

> `set_scope_config(scope_id, MemoryScopeConfig)` 会自动把配置加密后写入 KV 存储，并缓存到内存中；后续 `add_messages/search_user_mem/...` 会基于该 scope 的配置选择对应的大模型和嵌入模型。

### 配置 AgentMemoryConfig（变量记忆字段）

```python
from openjiuwen.core.memory import AgentMemoryConfig
from openjiuwen.core.common.schema.param import Param


agent_mem_cfg = AgentMemoryConfig(
    mem_variables=[
        Param.string("姓名", "用户姓名", required=False),
        Param.string("职业", "用户职业", required=False),
        Param.string("居住地", "用户居住地", required=False),
        Param.string("爱好", "用户爱好", required=False),
        Param.string("年龄", "用户年龄", required=False),
    ],
    enable_long_term_mem=True,
    enable_user_profile=True,
    enable_semantic_memory=True,
    enable_episodic_memory=True,
    enable_summary_memory=True
)
```

`AgentMemoryConfig` 会在调用 `add_messages` 时传入，用于指导记忆引擎从对话中抽取哪些变量和长期记忆。


## 写入消息并生成记忆（add_messages）

记忆引擎不再提供旧版的 `add_conversation_messages(user_id, group_id, messages, timestamp)` 接口，而是通过 `LongTermMemory.add_messages` 完成"写入消息 + 抽取记忆"：

```python
from datetime import datetime, timezone
from openjiuwen.core.foundation.llm.schema.message import BaseMessage


async def add_conversation(engine: LongTermMemory):
    user_id = "user1"
    scope_id = "app_demo_scope"
    session_id = "session_001"

    user_profile_messages = [
        BaseMessage(role="user", content="我叫张三，我喜欢打羽毛球"),
        BaseMessage(role="assistant", content="你好张三，很高兴认识你"),
        BaseMessage(role="user", content="我是一名软件工程师，住在杭州"),
    ]
    episodic_messages = [
        BaseMessage(role="user", content="昨天我去参加了杭州马拉松比赛，跑完全程用了4小时32分钟"),
        BaseMessage(role="assistant", content="太棒了！全程马拉松是个不小的挑战，4小时32分钟是很不错的成绩"),
        BaseMessage(role="user", content="是的，上周我还去北京出差，参加了一个技术峰会"),
    ]
    semantic_messages = [
        BaseMessage(role="user", content="Python是一种解释型编程语言，由Guido van Rossum在1991年创建"),
        BaseMessage(role="assistant", content="是的，Python以其简洁的语法和丰富的生态系统而闻名"),
        BaseMessage(role="user", content="杭州是浙江省的省会城市，也是中国重要的互联网产业中心"),
    ]

    timestamp = datetime.now(timezone.utc)
    for messages in [user_profile_messages, episodic_messages, semantic_messages]:
        await engine.add_messages(
            messages=messages,
            agent_config=agent_mem_cfg,
            user_id=user_id,
            scope_id=scope_id,
            session_id=session_id,
            timestamp=timestamp,
            gen_mem=True,                  # 是否生成长期记忆
            gen_mem_with_history_msg_num=5 # 生成记忆时使用多少条历史消息
        )
```
> 内部会自动：
> - 把消息写入消息表；
> - 结合历史消息与 `AgentMemoryConfig`，调用大模型进行变量/用户画像/情景记忆/语义记忆/摘要记忆抽取；
> - 把抽取出的长期记忆写入向量存储和 DB。

画像、情景、语义和摘要记忆类型说明：
- 用户画像记忆（user_profile）：关于用户本人的肯定或否定表述，包括基本身份、兴趣偏好、人际关系、资产状况等。例如："用户喜欢打羽毛球"
- 情景记忆（episodic_memory）：关于用户本人经历的具体事件的描述。例如："用户于2025年3月15日参加了杭州马拉松比赛"
- 语义记忆（semantic_memory）：其他信息片段，包括概念定义、事实信息、长期规则等的信息。例如："Python是一种解释型编程语言"
- 摘要记忆（summary_memory）：对用户与智能体的每次对话内容进行的概括总结。


## 查询变量记忆（get_variables）

使用 `get_variables` 可以按 `user_id + scope_id` 查询当前已提取的变量记忆：

```python
variables = await engine.get_variables(user_id="user1", scope_id="app_demo_scope")
print(variables)
# 可能输出：
# {"姓名": "张三", "职业": "软件工程师", "居住地": "杭州", "爱好": "打羽毛球", "年龄": "20"}
```

如果只想读取部分变量，可以传入 `names` 参数：

```python
name_only = await engine.get_variables(user_id="user1", scope_id="app_demo_scope", names=["姓名", "职业"])
```


## 分页查看长期记忆（get_user_mem_by_page）

对长期记忆可以通过分页接口查看，通过 `memory_type` 参数（memory_type类型：USER_PROFILE/EPISODIC_MEMORY/SEMANTIC_MEMORY）可以指定查看特定类型的记忆：

```python
from openjiuwen.core.memory.long_term_memory import MemoryType

# 查看指定类型的记忆
mem_list = await engine.get_user_mem_by_page(
    user_id="user1",
    scope_id="app_demo_scope",
    page_size=10,
    page_idx=1,
    memory_type=MemoryType.USER_PROFILE,  # 只看用户画像类记忆
)

for mem in mem_list:
    print(mem.mem_id, mem.type, mem.content)
```


## 语义检索记忆（search_user_mem）

`search_user_mem` 提供基于向量相似度的记忆检索能力：

```python
# search 用户画像记忆
search_results = await engine.search_user_mem(
    query="用户的职业是什么？",
    num=1,
    user_id="user1",
    scope_id="app_demo_scope",
    threshold=0.3,  # 过滤低于阈值的结果
)

for item in search_results:
    mem = item.mem_info
    print(f"mem_id={mem.mem_id}, type={mem.type}, score={item.score:.4f}, content={mem.content}")
```
```python
# search 用户情景记忆
search_results = await engine.search_user_mem(
    query="用户什么时候参加的马拉松？",
    num=1,
    user_id="user1",
    scope_id="app_demo_scope",
    threshold=0.3,  # 过滤低于阈值的结果
)

for item in search_results:
    mem = item.mem_info
    print(f"mem_id={mem.mem_id}, type={mem.type}, score={item.score:.4f}, content={mem.content}")
```
```python
# search 用户语义记忆
search_results = await engine.search_user_mem(
    query="Python是什么类型的语言",
    num=1,
    user_id="user1",
    scope_id="app_demo_scope",
    threshold=0.3,  # 过滤低于阈值的结果
)

for item in search_results:
    mem = item.mem_info
    print(f"mem_id={mem.mem_id}, type={mem.type}, score={item.score:.4f}, content={mem.content}")
```

返回值是若干 `MemResult`，每个包含：

- `mem_info: MemInfo`：包括 `mem_id/content/type`；
- `score: float`：相似度得分。

```python
    print(f"mem_id={mem.mem_id}, type={mem.type}, score={item.score:.4f}, content={mem.content}")
```

返回值是若干 `MemResult`，每个包含：

- `mem_info: MemInfo`：包括 `mem_id/content/type`；
- `score: float`：相似度得分。

## 统计与辅助查询

- **获取统计信息**：通过 `get_user_mem_by_page` / `user_mem_total_num` 可以统计某个用户在某个 scope 下的记忆条数。
