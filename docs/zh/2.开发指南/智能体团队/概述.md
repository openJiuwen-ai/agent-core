AgentTeams 是 Leader-Teammate 协作框架，通过 Leader（负责人）和 Teammates（队友）的协同工作完成复杂任务。相比直接使用 TeamRuntime，AgentTeams 提供了更高层次的抽象，支持跨进程执行、持久化状态和自动恢复，适用于需要进程隔离、长期运行或崩溃恢复的场景。

# 核心概念

## Leader（负责人）

- 负责任务分解、分配和协调
- 管理团队成员的生命周期
- 处理用户输入并分配给合适的队友

## Teammate（队友）

- 执行特定领域的任务
- 通过消息系统与 Leader 通信
- 可以独立运行在单独的进程中

## 架构组件

- **Transport（传输层）**: 负责智能体间消息传递（支持 `inprocess`、`pyzmq`）
- **Storage（存储层）**: 持久化团队状态、任务列表和消息（支持 `sqlite`、`postgresql`、`memory`）

# 团队生命周期

| 模式 | 说明 |
|------|------|
| Temporary（临时） | 任务完成后自动解散团队，适用于一次性任务 |
| Persistent（持久） | 团队状态和成员跨会话保留，支持断点续传和崩溃恢复 |

# 队友执行模式

| 模式 | 说明 |
|------|------|
| Plan Mode | 队友完成任务需要 Leader 批准，适用于需要严格控制的场景 |
| Build Mode | 队友直接完成任务，无需批准，适用于信任队友的场景（默认） |

# 队友启动模式

| 模式 | 说明 |
|------|------|
| Inprocess | 队友在同一进程内作为 asyncio 协程运行，适合单进程场景 |
| Process | 队友作为子进程运行，适合需要进程隔离的场景（默认） |

# 高级特性

- **健康检查与自动恢复**: Leader 定期检查队友进程状态，检测到不健康时自动重启
- **持久待命**: 持久团队每轮结束后进入待命状态，队友进程保持存活，下一次 `invoke()` 自动恢复
- **断点续传**: 支持通过 `resume_persistent_team()`（同进程）或 `recover_agent_team()`（崩溃恢复）恢复团队状态
- **用户 @mention**: 用户可通过 `@member_name message` 语法直接向特定队友发送消息，绕过 leader
- **预定义成员**: 预配置团队成员跳过动态 spawn，自动注册到数据库
- **跨进程通信**: 通过 pyzmq 支持跨进程、跨主机协作

# 适用场景

- 需要跨进程协作的复杂任务
- 长期运行的团队项目
- 需要持久化和恢复能力的场景
- 需要健康检查和自动恢复的生产环境

# 团队技能演进

团队技能演进是 AgentTeams 的延伸功能，支持从多 Agent 协作轨迹中提炼可复用的团队协作模式：

- **自动创建**: `TeamSkillCreateRail` 检测协作模式并建议创建团队技能
- **在线演进**: `TeamSkillRail` 支持轨迹分析、用户请求演进、PATCH 生成和审批
- **经验管理**: 经验评分、简化、重建机制确保团队技能持续优化

详见 [团队技能演进](./团队技能演进.md) 文档。
