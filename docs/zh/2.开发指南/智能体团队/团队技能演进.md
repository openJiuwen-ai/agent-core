# 团队技能演进

团队技能演进是 agent-core 提供的多 Agent 协作技能自动化创建和在线演进功能，支持从执行轨迹中提炼可复用的团队协作模式。

## 核心概念

### Team Skill

团队技能是一种特殊类型的 Skill，包含以下文件结构：

```
skills/<skill-name>/
├── SKILL.md          # 入口文件（YAML frontmatter + Markdown body）
│                      # frontmatter 必须包含 kind: team-skill
├── roles/
│   ├── <role-id>.md  # 每个角色的详细定义
│   └── ...
├── workflow.md       # 工作流描述（任务依赖、并行/串行关系）
├── bind.md           # 约束信息（轮数上限、超时、质量门）
└── evolutions.json   # 演进记录
```

与普通 Skill 不同，Team Skill：

- `kind: team-skill` 标识类型
- `roles` 列表定义角色及其 skills/tools 配置
- 适用于多 Agent 协作场景

### 演进机制

| 模块 | 功能 |
|------|------|
| `TeamSkillCreateRail` | 自动检测协作模式，建议创建团队技能 |
| `TeamSkillRail` | 在线演进：轨迹分析、用户请求、PATCH 生成/审批 |
| `TeamSkillOptimizer` | LLM 驱动的 PATCH 生成 |
| `ExperienceScorer` | 经验评分和简化维护 |
| `TeamTrajectoryAggregator` | 聚合团队成员轨迹 |

## 自动创建

### 原理

`TeamSkillCreateRail` 在 `AFTER_TASK_ITERATION` 回调中检测 `spawn_member` 调用次数。达到阈值时，注入 follow_up 提示，引导 Agent 通过 `ask_user` 确认并调用 `team-skill-creator` 技能。

### 配置示例

```python
from openjiuwen.harness.rails import TeamSkillCreateRail
from openjiuwen.harness import create_deep_agent

create_rail = TeamSkillCreateRail(
    skills_dir="/path/to/skills",
    language="cn",
    min_team_members_for_create=2,  # spawn_member >= 2 时触发
)

agent = create_deep_agent(
    model=model_client,
    system_prompt="你是一个团队 leader...",
    tools=team_tools,
    rails=[create_rail],
    enable_task_loop=True,
    workspace="/path/to/workspace",
)
```

### 创建流程

1. Agent 执行协作流程（调用 `build_team`、`spawn_member`、`create_task`）
2. Rail 检测 `spawn_member` 调用次数达到阈值
3. Rail 通过 `TaskLoopController.enqueue_follow_up()` 注入提示
4. Agent 调用 `ask_user` 工具向用户确认
5. 用户选择"创建"后，Agent 调用 `team-skill-creator` 技能
6. 技能根据轨迹生成 Team Skill 文件

## 在线演进

### TeamSkillRail 配置

```python
from openjiuwen.harness.rails import TeamSkillRail

team_rail = TeamSkillRail(
    skills_dir="/path/to/skills",
    llm=model_client,
    model="gpt-4",
    language="cn",
    auto_save=False,           # PATCH 需要用户审批
    async_evolution=True,      # 异步执行演进
    evolution_total_timeout_secs=600.0,
)

agent = create_deep_agent(
    model=model_client,
    tools=team_tools,
    rails=[team_rail],
    skills=["research-team"],  # 加载已有团队技能
)
```

### 触发时机

| 触发方式 | 触发条件 |
|----------|----------|
| 被动触发 | `view_task` 返回"所有任务已完成" |
| 主动触发 | 用户调用 `request_user_evolution()` |

### 演进路径

#### 1. 轨迹问题分析

Rail 分析团队执行轨迹，检测以下问题：

- **角色配合问题**: 协作断裂、数据未传递
- **约束违反**: 超时、产出格式不合规
- **流程低效**: 重复调用、多余步骤
- **角色能力不足**: 某角色多次失败或产出质量不达标

检测到问题时，Rail 调用 `TeamSkillOptimizer.generate_trajectory_patch()` 生成 PATCH。

#### 2. 用户请求演进

用户主动提供改进建议：

```python
request_id = await team_rail.request_user_evolution(
    skill_name="research-team",
    user_intent="增加 reviewer 角色，限制 research 时间不超过 10 分钟",
)
```

Rail 调用 `TeamSkillOptimizer.generate_user_patch()` 生成 PATCH。

### PATCH 审批流程

1. Rail 生成 PATCH 并暂存
2. PATCH 通过 `OutputSchema` 发送到 TUI
3. 用户选择"Accept"或"Reject"
4. 审批时调用 `on_approve_patch(request_id)` 写入 `evolutions.json`

```python
# 获取待审批事件
events = await team_rail.drain_pending_approval_events(wait=True)

for event in events:
    if event.type == "chat.ask_user_question":
        request_id = event.payload["request_id"]
        # 用户审批
        await team_rail.on_approve_patch(request_id)
```

## 经验管理

### 评分体系

ExperienceScorer 使用三维度加权评分：

| 维度 | 公式 | 权重 |
|------|------|------|
| E (Effectiveness) | Beta(1,1) 贝叶斯平滑 | 0.5 |
| U (Utilization) | `times_used / times_presented` | 0.3 |
| F (Freshness) | 时间衰减 + 版本过期惩罚 | 0.2 |

**综合评分**: `score = 0.5 * E + 0.3 * U + 0.2 * F`

### 简化操作

```python
result = await team_rail.request_simplify(
    skill_name="research-team",
    user_intent="清理过时经验",
)

# 返回: {"deleted": N, "merged": N, "refined": N, "kept": N, "errors": N}
```

| 操作 | 说明 |
|------|------|
| DELETE | 删除低质量经验（score < 阈值） |
| MERGE | 合并相似经验 |
| REFINE | 优化冗长内容 |
| KEEP | 保留高质量经验 |

### 重建流程

```python
prompt = await team_rail.request_rebuild(
    skill_name="research-team",
    user_intent="基于历史经验全面优化",
    min_score=0.5,  # 仅保留 score >= 0.5 的经验
)

# prompt 为重建指令文本，需注入到 Agent 执行
```

重建流程：

1. 归档当前 `SKILL.md` 和 `evolutions.json` 到 `archive/`
2. 筛选高质量演进经验（score >= min_score）
3. 构建重建提示，调用 `teamskill-creator` 技能生成新版本
4. 清空 `evolutions.json`

## 最佳实践

### 何时使用

- 多 Agent 协作场景（`spawn_member >= 2`）
- 需要沉淀协作经验、形成可复用模式
- 团队技能需要持续改进和优化

### 配置建议

| 配置项 | 建议 |
|--------|------|
| `min_team_members_for_create` | 根据团队规模调整，建议 2-3 |
| `auto_save` | 重要演进建议 `False`，确保审批 |
| `async_evolution` | 长任务建议 `True`，避免阻塞 |
| `evolution_total_timeout_secs` | 根据任务复杂度调整，建议 300-900 |

### 定期维护

1. 每周执行 `request_simplify()` 清理低质量经验
2. 每月评估是否需要 `request_rebuild()` 重建
3. 重要演进前设置 `auto_save=False` 确保用户审批

### 注意事项

- `TeamSkillCreateRail` 和 `TeamSkillRail` 可同时使用
- `TeamSkillCreateRail` 依赖 `enable_task_loop=True` 的 DeepAgent
- 演进生成的内容需用户审批才能生效
