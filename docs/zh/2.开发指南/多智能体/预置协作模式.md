# 预置协作模式多智能体团队

本章节介绍 openJiuwen 框架内置的三种多智能体协作模式：**层次化模式（Hierarchical）** 和 **交接模式（Handoff）**。这些预置模式封装了常见的多智能体协作场景，开箱即用。

## 概述

openJiuwen 提供了以下预置协作模式：

| 模式 | 适用场景 | 通信方式 | 特点 |
|------|----------|----------|------|
| **Hierarchical Tools** | 层次化任务分解 | Agent as Tool | 通过 ability_manager 将子 Agent 注册为工具 |
| **Hierarchical MessageBus** | 层次化任务分解 | P2P MessageBus | 通过消息总线进行 P2P 通信，支持并行调用 |
| **Handoff** | 顺序协作、任务转交 | Pub-Sub + 状态机 | Agent 间通过工具主动转交任务，支持中断恢复 |

## 层次化模式（Hierarchical）

层次化模式适用于**自上而下的任务分解场景**，上层 Agent 将复杂任务分解为子任务，委托给下层 Agent 执行。

### 两种实现方式

openJiuwen 提供两种层次化模式实现：

#### 1. Hierarchical Tools 模式

**核心机制**：通过 `ability_manager` 将子 Agent 注册为工具，上层 Agent 通过工具调用的方式调用子 Agent。

**特点**：
- 简单直观，与普通工具调用一致
- 子 Agent 作为工具暴露给 LLM
- 适合层次清晰、调用关系固定的场景

**使用场景**：
- 研究团队：主管 → 文献调研员 / 数据分析员 → 统计专家
- 软件开发：架构师 → 前端开发 / 后端开发 → 测试工程师

#### 2. Hierarchical MessageBus 模式

**核心机制**：通过 P2P MessageBus 进行通信，Supervisor Agent 使用 `P2PAbilityManager` 管理子 Agent，支持并行调用。

**特点**：
- 支持并行调用多个子 Agent（通过 `max_parallel_sub_agents` 控制并发数）
- 基于消息总线，解耦性更强
- 适合需要并行处理、动态调度的场景

**使用场景**：
- 数据处理流水线：协调器 → 多个数据处理节点（并行）
- 分布式任务调度：调度器 → 多个工作节点（并行）

### 核心组件

#### HierarchicalTeamConfig

配置类，定义层次化团队的参数：

**Hierarchical Tools 模式**：
```python
from openjiuwen.core.multi_agent.teams.hierarchical_tools import HierarchicalTeamConfig

config = HierarchicalTeamConfig(
    root_agent=root_agent_card,  # 根节点 Agent（必填）
)
```

**Hierarchical MessageBus 模式**：
```python
from openjiuwen.core.multi_agent.teams.hierarchical_msgbus import HierarchicalTeamConfig

config = HierarchicalTeamConfig(
    supervisor_agent=supervisor_card,  # Supervisor Agent（必填）
)
```

#### HierarchicalTeam

团队类，封装层次化协作逻辑：

**Hierarchical Tools 模式**：
```python
from openjiuwen.core.multi_agent.teams.hierarchical_tools import HierarchicalTeam

team = HierarchicalTeam(card=team_card, config=config)

# 注册 Agent 并指定父子关系
team.add_agent(root_card, root_provider)
team.add_agent(child_card, child_provider, parent_agent_id="root_id")
```

**Hierarchical MessageBus 模式**：
```python
from openjiuwen.core.multi_agent.teams.hierarchical_msgbus import HierarchicalTeam

team = HierarchicalTeam(card=team_card, config=config)

# 注册所有 Agent
team.add_agent(supervisor_card, supervisor_provider)
team.add_agent(sub_agent_card, sub_agent_provider)
```

#### SupervisorAgent（仅 MessageBus 模式）

内置的 Supervisor Agent，结合 ReActAgent 和 P2PAbilityManager：

```python
from openjiuwen.core.multi_agent.teams.hierarchical_msgbus import SupervisorAgent

supervisor_card, supervisor_provider = SupervisorAgent.create(
    agents=[sub_agent1_card, sub_agent2_card],  # 子 Agent 列表
    model_client_config=model_client_config,
    model_request_config=model_request_config,
    agent_card=supervisor_card,
    system_prompt="你是协调器，负责分配任务给子 Agent",
    max_iterations=5,
    max_parallel_sub_agents=10,  # 最大并行调用数
)
```

### 完整示例

#### 示例 1：Hierarchical Tools 模式 - 三层研究团队

**场景**：研究主管协调文献调研员和数据分析员，数据分析员可调用统计专家。

**团队结构**：
```
研究主管 (Research Director)
├── 文献调研员 (Literature Researcher)
└── 数据分析员 (Data Analyst)
    └── 统计专家 (Statistics Expert)
```

**完整代码**：[hierarchical_tools_research_team.py](../../examples/multi_agent/builtin_teams/hierarchical_tools_research_team.py)

**核心代码片段**：

```python
from openjiuwen.core.multi_agent.teams.hierarchical_tools import (
    HierarchicalTeam,
    HierarchicalTeamConfig,
)

# 1. 创建团队
team_config = HierarchicalTeamConfig(root_agent=research_director_card)
team = HierarchicalTeam(card=team_card, config=team_config)

# 2. 注册 Agent 并指定父子关系
team.add_agent(research_director_card, lambda: research_director)  # 第一层
team.add_agent(
    literature_researcher_card,
    lambda: literature_researcher,
    parent_agent_id="research_director"  # 第二层
)
team.add_agent(
    data_analyst_card,
    lambda: data_analyst,
    parent_agent_id="research_director"  # 第二层
)
team.add_agent(
    statistics_expert_card,
    lambda: statistics_expert,
    parent_agent_id="data_analyst"  # 第三层
)

# 3. 运行团队
result = await team.invoke({
    "query": "请研究人工智能在医疗诊断中的应用"
})
```

**运行效果**：
1. 研究主管接收任务，决策调用文献调研员和数据分析员
2. 文献调研员检索相关文献
3. 数据分析员分析数据，必要时调用统计专家
4. 研究主管汇总结果并返回

#### 示例 2：Hierarchical MessageBus 模式 - 三层研究团队

**场景**：与示例 1 相同，但使用 MessageBus 模式实现。

**完整代码**：[hierarchical_msgbus_research_team.py](../../examples/multi_agent/builtin_teams/hierarchical_msgbus_research_team.py)

**核心代码片段**：

```python
from openjiuwen.core.multi_agent.teams.hierarchical_msgbus import (
    HierarchicalTeam,
    HierarchicalTeamConfig,
    SupervisorAgent,
)

# 1. 创建 Supervisor Agent（数据分析员）
data_analyst_card, data_analyst_provider = SupervisorAgent.create(
    agents=[statistics_expert_card],  # 管理统计专家
    model_client_config=model_client_config,
    model_request_config=model_request_config,
    agent_card=data_analyst_card,
    system_prompt="你是数据分析员，可以调用统计专家",
    max_parallel_sub_agents=5
)

# 2. 创建根 Supervisor Agent（研究主管）
research_director_card, research_director_provider = SupervisorAgent.create(
    agents=[literature_researcher_card, data_analyst_card],
    model_client_config=model_client_config,
    model_request_config=model_request_config,
    agent_card=research_director_card,
    system_prompt="你是研究主管，可以调用文献调研员和数据分析员",
    max_parallel_sub_agents=5
)

# 3. 创建团队并注册所有 Agent
team_config = HierarchicalTeamConfig(supervisor_agent=research_director_card)
team = HierarchicalTeam(card=team_card, config=team_config)

team.add_agent(research_director_card, research_director_provider)
team.add_agent(literature_researcher_card, lambda: literature_researcher)
team.add_agent(data_analyst_card, data_analyst_provider)
team.add_agent(statistics_expert_card, lambda: statistics_expert)

# 4. 运行团队
result = await team.invoke({
    "query": "请研究人工智能在医疗诊断中的应用"
})
```

**运行效果**：与示例 1 相同，但通过 MessageBus 实现，支持并行调用。

### 两种模式对比

| 特性 | Hierarchical Tools | Hierarchical MessageBus |
|------|-------------------|------------------------|
| **实现方式** | ability_manager | P2P MessageBus |
| **并行调用** | 不支持 | 支持（max_parallel_sub_agents） |
| **配置复杂度** | 简单 | 中等 |
| **解耦性** | 中等 | 高 |
| **适用场景** | 层次清晰、串行调用 | 需要并行处理、动态调度 |

**选择建议**：
- 简单层次化任务 → Hierarchical Tools
- 需要并行调用多个子 Agent → Hierarchical MessageBus
- 需要动态调度、高解耦 → Hierarchical MessageBus

## 交接模式（Handoff）

交接模式适用于**顺序协作场景**，Agent 之间通过主动转交任务的方式协作，每个 Agent 可以决定是完成任务还是转交给其他 Agent。

### 核心机制

- **工具驱动**：每个 Agent 自动注入 `transfer_to_{target}` 工具
- **状态机管理**：`HandoffOrchestrator` 管理转交状态和路由规则
- **中断恢复**：支持任务中断和恢复（通过 Session 持久化）

### 核心组件

#### HandoffConfig

配置交接规则：

```python
from openjiuwen.core.multi_agent.teams.handoff import HandoffConfig, HandoffRoute

config = HandoffConfig(
    start_agent=triage_card,  # 起始 Agent
    max_handoffs=10,  # 最大转交次数
    routes=[  # 转交路由规则（空列表表示全连接）
        HandoffRoute(source="agent_a", target="agent_b"),
        HandoffRoute(source="agent_b", target="agent_c"),
    ],
    termination_condition=None,  # 可选的终止条件函数
)
```

#### HandoffTeam

交接团队类：

```python
from openjiuwen.core.multi_agent.teams.handoff import HandoffTeam, HandoffTeamConfig

team_config = HandoffTeamConfig(handoff=config)
team = HandoffTeam(card=team_card, config=team_config)

# 注册 Agent
team.add_agent(agent_a_card, agent_a_provider)
team.add_agent(agent_b_card, agent_b_provider)

# 运行
result = await team.invoke({"query": "用户问题"})
```

#### HandoffTool

自动注入的转交工具，Agent 通过调用此工具转交任务：

```python
# Agent 的 LLM 会看到以下工具：
# - transfer_to_agent_b: 转交任务给 agent_b
# - transfer_to_agent_c: 转交任务给 agent_c

# LLM 调用示例：
{
    "name": "transfer_to_agent_b",
    "arguments": {
        "reason": "这是技术问题，需要技术支持处理",
        "message": "用户反馈登录失败"
    }
}
```

### 完整示例

#### 示例 3：Handoff 模式 - 客服系统

**场景**：客服分流系统，根据用户问题类型转交给技术支持或账单支持。

**Agent 流程**：
```
Triage Agent (分流)
├→ Technical Support (技术支持)
└→ Billing Support (账单支持)
   ↔ (技术支持和账单支持可互相转交)
```

**完整代码**：[handoff_customer_service.py](../../examples/multi_agent/builtin_teams/handoff_customer_service.py)

**核心代码片段**：

```python
from openjiuwen.core.multi_agent.teams.handoff import (
    HandoffTeam,
    HandoffTeamConfig,
    HandoffConfig,
    HandoffRoute,
)

# 1. 配置 handoff 路由规则
handoff_config = HandoffConfig(
    start_agent=triage_card,  # 从分流 Agent 开始
    max_handoffs=5,
    routes=[
        # 分流 Agent 可以转交给技术支持或账单支持
        HandoffRoute(source="triage_agent", target="technical_support"),
        HandoffRoute(source="triage_agent", target="billing_support"),
        # 技术支持和账单支持可以互相转交
        HandoffRoute(source="technical_support", target="billing_support"),
        HandoffRoute(source="billing_support", target="technical_support"),
    ]
)

# 2. 创建团队
team_config = HandoffTeamConfig(handoff=handoff_config)
team = HandoffTeam(card=team_card, config=team_config)

# 3. 注册 Agent
team.add_agent(triage_card, lambda: triage_agent)
team.add_agent(technical_support_card, lambda: technical_support)
team.add_agent(billing_support_card, lambda: billing_support)

# 4. 运行
result = await team.invoke({"query": "我的账单怎么这么贵？"})
```

**运行效果**：
1. Triage Agent 分析问题类型
2. 识别为账单问题，调用 `transfer_to_billing_support`
3. Billing Support Agent 处理账单问题并返回结果

**Agent 系统提示词示例**：

```python
# Triage Agent
"你是客服分流专员。分析用户问题：\n"
"- 技术问题（产品使用、故障、功能）→ 转交 technical_support\n"
"- 账单问题（付款、退款、发票）→ 转交 billing_support\n"
"- 简单问候或感谢 → 直接回复\n"
"使用 transfer_to_xxx 工具转交任务。"

# Technical Support Agent
"你是技术支持专员，负责解决产品技术问题。"
"提供详细的故障排查步骤和解决方案。"
"如果问题超出技术范围（如账单问题），转交 billing_support。"

# Billing Support Agent
"你是账单支持专员，负责处理账单、付款、退款问题。"
"提供清晰的账单说明和付款指引。"
"如果问题是技术问题，转交 technical_support。"
```

### 高级特性

#### 1. 中断与恢复

Handoff 模式支持任务中断和恢复：

```python
# Agent 返回中断信号
return {"result_type": "interrupt", "message": "需要用户确认"}

# 或抛出 AgentInterrupt 异常
from openjiuwen.core.session.interaction.base import AgentInterrupt
raise AgentInterrupt("需要用户输入")

# 恢复执行
result = await team.invoke({"query": "继续"}, session=previous_session)
```

#### 2. 自定义终止条件

```python
def custom_termination(orchestrator):
    # 当转交次数超过 3 次时终止
    return orchestrator.handoff_count > 3

config = HandoffConfig(
    start_agent=start_card,
    termination_condition=custom_termination
)
```

#### 3. 全连接路由

当 `routes` 为空列表时，任意 Agent 可以转交给任意其他 Agent：

```python
config = HandoffConfig(
    start_agent=start_card,
    routes=[],  # 全连接
)
```

## 最佳实践

### 1. 选择合适的协作模式

| 场景 | 推荐模式 |
|------|----------|
| 自上而下任务分解，层次清晰 | Hierarchical Tools |
| 需要并行调用多个子 Agent | Hierarchical MessageBus |
| Agent 间顺序协作，需要灵活转交 | Handoff |
| 需要中断恢复、状态持久化 | Handoff |

### 2. 设计清晰的 Agent 职责

每个 Agent 应有明确的职责边界：

```python
# ✅ 好的设计
"你是技术支持专员，负责解决产品技术问题"

# ❌ 不好的设计
"你是客服专员，负责处理所有用户问题"
```

### 3. 合理配置转交规则

- **Hierarchical 模式**：明确父子关系，避免循环依赖
- **Handoff 模式**：设置合理的 `max_handoffs`，避免无限循环

### 4. 编写高质量的系统提示词

系统提示词应明确说明：
- Agent 的职责范围
- 可调用的子 Agent / 可转交的目标
- 何时调用 / 转交

### 5. 处理异常情况

```python
try:
    result = await team.invoke(inputs)
except Exception as e:
    # 处理团队执行异常
    logger.error(f"Team execution failed: {e}")
```

## 常见问题

### Q1: Hierarchical Tools 和 MessageBus 模式有什么区别？

**A**: 主要区别在于通信方式和并行能力：
- **Tools 模式**：通过 ability_manager 注册子 Agent 为工具，串行调用
- **MessageBus 模式**：通过 P2P 消息总线通信，支持并行调用（max_parallel_sub_agents）

### Q2: Handoff 模式如何避免无限循环？

**A**: 通过以下机制：
1. `max_handoffs` 限制最大转交次数
2. `termination_condition` 自定义终止条件
3. `routes` 限制转交路由，避免循环路径

### Q3: 如何调试多智能体协作？

**A**: 
1. 启用日志：`logger.setLevel(logging.DEBUG)`
2. 查看 handoff_history（Handoff 模式）
3. 使用 `stream()` 方法实时查看中间结果

### Q4: 可以混合使用多种模式吗？

**A**: 可以。例如：
- Hierarchical 团队的某个子 Agent 可以是 Handoff 团队
- Handoff 团队的某个 Agent 可以是 Hierarchical 团队
